<div id="verceljson-file">
  ## vercel.json-Datei
</div>

Die Datei `vercel.json` ist die Konfigurationsdatei von Vercel, mit der Sie festlegen, wie Ihr Projekt gebaut und bereitgestellt wird. Sie liegt im Stammverzeichnis Ihres Projekts und steuert verschiedene Aspekte Ihrer Bereitstellung, darunter Routing, Weiterleitungen, Header und Build-Einstellungen.

Wir verwenden die Einstellung `rewrites`, um Anfragen von Ihrer Hauptdomain zu Ihrer Dokumentation zu proxen.

Rewrites ermöglichen es, eingehende Anfragen auf andere Ziele abzubilden, ohne die URL im Browser zu ändern. Wenn jemand `yoursite.com/docs` aufruft, lädt Vercel intern Inhalte von `your-subdomain.mintlify.dev/docs`, während der Nutzer weiterhin `yoursite.com/docs` im Browser sieht. Das unterscheidet sich von Weiterleitungen, die Nutzer vollständig auf eine andere URL schicken würden.

<div id="configuration">
  ## Konfiguration
</div>

Um Ihre Dokumentation unter dem benutzerdefinierten Unterpfad `/docs` mit Vercel zu hosten, fügen Sie die folgende Konfiguration zu Ihrer `vercel.json` hinzu:

```json
{
  "rewrites": [
    {
      "source": "/docs",
      "destination": "https://[subdomain].mintlify.dev/docs"
    },
    {
      "source": "/docs/:match*",
      "destination": "https://[subdomain].mintlify.dev/docs/:match*"
    }
  ]
}
```

- **`source`**: Das Pfadmuster auf Ihrer Domain, das die Umschreibung auslöst.
- **`destination`**: Das Ziel, an das die Anfrage weitergeleitet wird.
- **`:match*`**: Ein Platzhalter, der alle Pfadsegmente nach `/docs/` erfasst.

Weitere Informationen finden Sie unter [Configuring projects with vercel.json: Rewrites](https://vercel.com/docs/projects/project-configuration#rewrites) in der Vercel-Dokumentation.

<div id="using-external-proxies-with-vercel">
  ## Externe Proxys mit Vercel verwenden
</div>

Wenn Sie einen externen Proxy (wie Cloudflare oder AWS CloudFront) vor Ihrer Vercel-Bereitstellung einsetzen, müssen Sie ihn korrekt konfigurieren, um Konflikte mit der Domainverifizierung von Vercel und der Ausstellung von SSL-Zertifikaten zu vermeiden.

Eine fehlerhafte Proxy-Konfiguration kann verhindern, dass Vercel Let's-Encrypt-SSL-Zertifikate ausstellt, und zu Fehlern bei der Domainverifizierung führen.

Siehe die [unterstützten Anbieter](https://vercel.com/guides/how-to-setup-verified-proxy#supported-providers-verified-proxy-lite) in der Vercel-Dokumentation.

<div id="required-path-allowlist">
  ### Erforderliche Pfad-Allowlist
</div>

Ihr externer Proxy muss den Traffic zu diesen spezifischen Pfaden zulassen, ohne zu blockieren, umzuleiten oder aggressiv zu cachen:

- `/.well-known/acme-challenge/*` – erforderlich für die Let's-Encrypt-Zertifikatsüberprüfung
- `/.well-known/vercel/*` – erforderlich für die Vercel-Domainverifizierung

Diese Pfade sollten unverändert direkt an Ihre Vercel-Bereitstellung durchgereicht werden.

<div id="header-forwarding-requirements">
  ### Anforderungen an die Header-Weiterleitung
</div>

Stellen Sie sicher, dass Ihr Proxy den `HOST`-Header korrekt weiterleitet. Ohne korrekte Header-Weiterleitung schlagen Verifizierungsanfragen fehl.

<div id="testing-your-proxy-setup">
  ### Testen Ihrer Proxy-Konfiguration
</div>

So überprüfen Sie, ob Ihr Proxy korrekt konfiguriert ist:

1. Prüfen Sie, dass `https://[yourdomain].com/.well-known/vercel/` eine Antwort liefert.
2. Stellen Sie sicher, dass SSL-Zertifikate in Ihrem Vercel-Dashboard korrekt ausgestellt werden.
3. Vergewissern Sie sich, dass die Domainverifizierung erfolgreich abgeschlossen wird.
