<Info>
  [Pro- und Enterprise-Pläne](https://mintlify.com/pricing?ref=docs-ci) enthalten CI-Prüfungen für GitHub-Repositories.
</Info>

Verwenden Sie CI-Prüfungen, um Ihre Docs zu linten, Fehler zu erkennen und vor dem Deployment Warnungen auszugeben. CI-Prüfungen sind so konfiguriert, dass sie bei Commits in den konfigurierten Deployment-Branch oder bei Pull Requests gegen diesen Branch ausgeführt werden.

<div id="installation">
  ## Installation
</div>

Beginnen Sie, indem Sie den Schritten auf der Seite [GitHub](/de/settings/github) folgen.

Bei GitHub-Apps können Sie die Berechtigungen auf ein einzelnes Repository beschränken. Wir empfehlen dies nachdrücklich, da wir nur Zugriff auf das Repository benötigen, in dem Ihre Dokumentation gehostet wird.

<div id="configuration">
  ## Konfiguration
</div>

Konfigurieren Sie die in der CI für eine Bereitstellung aktivierten Prüfungen, indem Sie in Ihrem Dashboard zur Seite [Add-ons](https://dashboard.mintlify.com/products/addons) navigieren. Aktivieren Sie die Prüfungen, die Sie ausführen möchten.

Beim Aktivieren der Prüfungen können Sie wählen, ob sie auf der Stufe `Warning` oder `Blocking` ausgeführt werden.

- Eine Prüfung auf der Stufe `Warning` führt niemals zu einem Fehlerstatus, selbst wenn es Fehler oder Vorschläge gibt.
- Eine Prüfung auf der Stufe `Blocking` führt zu einem Fehlerstatus, wenn sie nicht bestanden wird oder Änderungen vorgeschlagen werden.

<div id="available-ci-checks">
  ## Verfügbare CI-Checks
</div>

<div id="broken-links">
  ### Kaputte Links
</div>

Ähnlich wie der [CLI-Link-Checker](/de/settings/broken-links#broken-links) auf Ihrem lokalen Rechner prüfen wir Ihre Dokumentation automatisch auf kaputte Links.

Um die Ergebnisse dieses Checks zu sehen, öffnen Sie auf GitHub die Ergebnisseite der Checks für einen bestimmten Commit.

<div id="vale">
  ### Vale
</div>

[Vale](https://vale.sh/) ist ein Open-Source-, regelbasierter Prosa-Linter, der eine Reihe von Dokumenttypen unterstützt, darunter Markdown und MDX.

Mintlify unterstützt das automatische Ausführen von Vale in einem CI-Check und zeigt die Ergebnisse als Check-Status an.

<div id="configuration">
  #### Konfiguration
</div>

Wenn Sie eine Datei `.vale.ini` im Wurzelverzeichnis der Inhalte Ihrer Bereitstellung haben, verwenden wir diese Konfigurationsdatei automatisch. Wir verwenden außerdem automatisch alle Konfigurationsdateien in Ihrem angegebenen `stylesPath`.

Wenn Sie keine Vale-Konfigurationsdatei haben, wird die Standardkonfiguration automatisch geladen.

```mdx Default vale.ini configuration expandable
# Top level styles
StylesPath = /app/styles
MinAlertLevel = suggestion
IgnoredScopes = code, tt, img, url, a
SkippedScopes = script, style, pre, figure, code

# Vocabularies
Vocab = Mintlify

# This is required since Vale doesn't officially support MDX
[formats]
mdx = md

# MDX support
[*.mdx]
BasedOnStyles = Vale
Vale.Terms = NO # Enforces really harsh capitalization rules, keep off

# `import ...`, `export ...`
# `<Component ... />`
# `<Component>...</Component>`
# `{ ... }`
TokenIgnores = (?sm)((?:import|export) .+?$), \
(?<!`)(<\w+ ?.+ ?\/>)(?!`), \
(<[A-Z]\w+>.+?<\/[A-Z]\w+>)

# Exclude:
# `<Component \n ... />`
BlockIgnores = (?sm)^(<\w+\n .*\s\/>)$, \
(?sm)^({.+.*})

CommentDelimiters = {/*, */}
```

<Warning>
  Bitte beachten Sie, dass wir aus Sicherheitsgründen keine absoluten `stylesPath`-Angaben oder `stylesPath` mit `..` unterstützen können. Verwenden Sie relative Pfade und legen Sie den `stylesPath` in Ihrem Repository ab.

<div id="packages">
  #### Pakete
</div>

Vale unterstützt eine Reihe von [Paketen](https://vale.sh/docs/keys/packages), mit denen Sie Rechtschreib- und Stilfehler prüfen können.
Alle Pakete, die Sie in Ihrem Repository unter dem korrekten `stylesPath` ablegen, werden automatisch installiert und in Ihrer Vale-Konfiguration verwendet.

Für Pakete, die nicht in Ihrem Repository enthalten sind, können Sie beliebige Pakete aus dem [Vale-Paketregister](https://vale.sh/explorer) angeben; diese werden automatisch heruntergeladen und in Ihrer Vale-Konfiguration verwendet.

<Warning>
  Bitte beachten Sie, dass wir aus Sicherheitsgründen das automatische Herunterladen von Paketen, die nicht aus dem [Vale-Paketregister](https://vale.sh/explorer) stammen, nicht unterstützen können.
</Warning>

<div id="vale-with-mdx">
  #### Vale mit `MDX`
</div>

Vale unterstützt `MDX` nicht nativ, aber der Autor von Vale stellt eine [benutzerdefinierte Erweiterung](https://github.com/errata-ai/MDX) dafür bereit.

Wenn Sie diese Erweiterung nicht verwenden möchten, empfehlen wir die folgenden Zeilen in Ihrer Datei `.vale.ini`:

```ini
[formats]
mdx = md

[*.mdx]
CommentDelimiters = {/*, */}

TokenIgnores = (?sm)((?:import|export) .+?$), \
(?<!`)(<\w+ ?.+ ?\/>)(?!`), \
(<[A-Z]\w+>.+?<\/[A-Z]\w+>)

BlockIgnores = (?sm)^(<\w+\n .*\s\/>)$, \
(?sm)^({.+.*})
```

Um die in Dokumenten verwendeten Kommentare von Vale zu nutzen, verwenden Sie MDX‑Kommentare `{/* ... */}`. Wenn Sie in Ihrer Konfiguration die Einstellung `CommentDelimiters = {/*, */}` verwenden, interpretiert Vale diese Kommentare beim Linting automatisch. Das bedeutet, Sie können die integrierten Funktionen von Vale wie das Überspringen von Zeilen oder Abschnitten unkompliziert verwenden. Weitere Informationen finden Sie in den [Einstellungen](https://vale.sh/docs/keys/commentdelimiters).

```mdx
{/* vale off */}

Dieser Text wird von Vale ignoriert

{/* vale on */}
```

Wenn Sie sich entscheiden, `CommentDelimiters` nicht zu verwenden, aber weiterhin Vale‑Kommentare zu nutzen, müssen Sie alle Vale‑Kommentare in MDX‑Kommentare `{/* ... */}` einschließen. Zum Beispiel:

```mdx
{/* <!-- vale off --> */}

Dieser Text wird von Vale ignoriert

{/* <!-- vale on --> */}
```

Diese Kommentar-Tags werden innerhalb von Mintlify-Komponenten nicht unterstützt, können jedoch auf der Basisebene eines Dokuments überall verwendet werden.
