Mintlify berücksichtigt automatisch viele SEO-Best Practices, darunter:

- Generierung von Metatags
- Erstellung von Sitemap und `robots.txt`
- Semantische HTML-Struktur
- Optimierung für Mobilgeräte

Sie können die Metatags Ihrer Website vollständig anpassen, indem Sie das Feld `metatags` zu Ihrer `docs.json` oder zum Frontmatter einer Seite hinzufügen.

<div id="global-meta-tags">
  ## Globale Meta-Tags
</div>

Sie können Standard-Meta-Tags festlegen, indem Sie das Feld `metadata` zu Ihrer `docs.json` hinzufügen.

```json
"seo": {
    "metatags": {
        "og:image": "link to your default meta tag image"
    }
}
```

<div id="page-specific-meta-tags">
  ## Seitenspezifische Meta-Tags
</div>

Fügen Sie Ihre Meta-Tags wie folgt in die Metadaten der Seite ein:

```mdx
---
title: 'Your Example Page Title'
'og:image': 'link to your meta tag image'
---
```

<Warning>
  Meta-Tags mit Doppelpunkten müssen in Anführungszeichen gesetzt werden, wenn Sie sie in die Seitenmetadaten einfügen.
</Warning>

<div id="sitemaps-and-robotstxt-files">
  ## Sitemaps und robots.txt-Dateien
</div>

Mintlify generiert automatisch eine `sitemap.xml`-Datei und eine `robots.txt`-Datei. Sie können Ihre Sitemap anzeigen, indem Sie `/sitemap.xml` an die URL Ihrer Dokumentationsseite anhängen.

Standardmäßig werden nur Seiten berücksichtigt, die in Ihrer `docs.json` aufgeführt sind. Um versteckte Links einzubeziehen, fügen Sie `seo.indexing` zu Ihrer `docs.json` hinzu:

```json
"seo": {
    "indexing": all
}
```

<div id="custom-sitemaps-and-robotstxt-files">
  ### Benutzerdefinierte Sitemaps und robots.txt-Dateien
</div>

Um eine benutzerdefinierte `sitemap.xml`- oder `robots.txt`-Datei hinzuzufügen, erstellen Sie eine `sitemap.xml`- oder `robots.txt`-Datei im Root-Verzeichnis Ihres Projekts. Das Hinzufügen einer benutzerdefinierten Datei überschreibt die automatisch generierte Datei mit demselben Namen. Wenn Sie eine benutzerdefinierte Datei löschen, wird wieder die Standarddatei verwendet.

<div id="disabling-indexing">
  ## Deaktivieren der Indexierung
</div>

Wenn du verhindern möchtest, dass eine Seite von Suchmaschinen indexiert wird, kannst du Folgendes in das [Frontmatter](de/pages) deiner Seite aufnehmen:

```
---
noindex: true
---
```

Du kannst `noindex` auch für alle Seiten in deiner Dokumentation festlegen, indem du in deiner `docs.json` das Feld `metadata.robots` auf `"noindex"` setzt:

```json
"seo": {
    "metatags": {
      "robots": "noindex"
    }
  }
```

<div id="seo-best-practices">
  ## SEO-Best Practices
</div>

<AccordionGroup>
  <Accordion title="Aussagekräftige Titel und Beschreibungen verfassen">
    - Klare, aussagekräftige Seitentitel verwenden (50–60 Zeichen)
    - Prägnante, überzeugende Beschreibungen schreiben (150–160 Zeichen)
    - Relevante Keywords einbinden
    - Für jede Seite einzigartige Titel und Beschreibungen erstellen
  </Accordion>

  <Accordion title="Inhaltsstruktur optimieren">
    - Korrekte Überschriftenhierarchie nutzen (H1 → H2 → H3)
    - Zuerst für Menschen schreiben, dann für Suchmaschinen
    - Relevante Keywords in Überschriften und Inhalt integrieren
    - URLs kurz, aussagekräftig und hierarchisch organisiert halten
    - Lange Inhalte mit Zwischenüberschriften und Listen gliedern
  </Accordion>

  <Accordion title="Interne Verlinkungsstrategie">
    - Auf thematisch verwandte Seiten innerhalb der Dokumentation verlinken
    - Aussagekräftigen Linktext statt „hier klicken“ verwenden
    - Themen-Cluster bilden, indem verwandte Konzepte verlinkt werden
    - Die automatischen Querverweisfunktionen nutzen
  </Accordion>

  <Accordion title="Bild-SEO">
    - Aussagekräftige Dateinamen für Bilder verwenden
    - Immer Alt-Text für Barrierefreiheit und SEO hinzufügen
    - Bildgrößen für schnelleres Laden optimieren
    - Relevante Bilder nutzen, die den Inhalt unterstützen
  </Accordion>
</AccordionGroup>
