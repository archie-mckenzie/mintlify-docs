Wenn Sie derzeit einzelne `MDX`-Seiten für Ihre API-Endpunkte verwenden, können Sie auf die automatische Generierung von Seiten aus Ihrer OpenAPI-Spezifikation umsteigen und dabei die Anpassbarkeit der einzelnen Seiten beibehalten. So reduzieren Sie den Pflegeaufwand und erhöhen die Konsistenz Ihrer API-Dokumentation.

Sie können Metadaten und Inhalte für jeden Endpunkt in Ihrer OpenAPI-Spezifikation definieren und die Endpunkte in der Navigation nach Wunsch anordnen.

<div id="cli-migration">
  ## CLI-Migration
</div>

Der Befehl `mint migrate-mdx` ist die empfohlene Methode, um von MDX-Endpunktseiten zu automatisch generierten Seiten zu migrieren.

Dieser Befehl:

- Analysiert die Navigationsstruktur deiner `docs.json`.
- Identifiziert MDX-Seiten, die OpenAPI-Endpunktseiten generieren.
- Extrahiert Inhalte aus MDX-Dateien und verschiebt sie in die Erweiterung `x-mint` in deiner OpenAPI-Spezifikation.
- Aktualisiert deine `docs.json`, damit die OpenAPI-Endpunkte direkt statt über MDX-Dateien referenziert werden.
- Löscht die ursprünglichen MDX-Endpunktdateien.

<Info>
  Wenn für einen Endpunkt bereits `x-mint` definiert ist und es zusätzlich eine MDX-Seite mit Inhalt für diesen Endpunkt gibt, überschreibt der MDX-Inhalt die vorhandenen `x-mint`-Einstellungen.

  Wenn es mehrere MDX-Seiten für denselben Endpunkt mit unterschiedlichem Inhalt gibt, verwendet das Skript den Inhalt der Seite, die in deiner `docs.json` zuletzt aufgeführt ist.

  Das Migrationstool unterstützt keine Vorschau der Änderungen, bevor sie angewendet werden.
</Info>

<Steps>
  <Step title="Bereite deine OpenAPI-Spezifikation vor.">
    Stelle sicher, dass deine OpenAPI-Spezifikation gültig ist und alle Endpunkte enthält, die du dokumentieren möchtest.

    Alle MDX-Seiten, die du migrieren möchtest, müssen im Frontmatter einen Eintrag `openapi:` haben, der auf einen Endpunkt verweist.

    <Tip>
      Validiere deine OpenAPI-Datei mit dem [Swagger Editor](https://editor.swagger.io/) oder der [Mint CLI](https://www.npmjs.com/package/mint).
    </Tip>
  </Step>

  <Step title="Installiere die Mint CLI">
    Installiere oder aktualisiere bei Bedarf die [Mint CLI](/de/installation).
  </Step>

  <Step title="Führe den Migrationsbefehl aus.">
    ```bash
    mint migrate-mdx
    ```
  </Step>
</Steps>

<div id="manual-migration-steps">
  ## Manuelle Migrationsschritte
</div>

<Steps>
  <Step title="Bereiten Sie Ihre OpenAPI-Spezifikation vor.">
    Stellen Sie sicher, dass Ihre OpenAPI-Spezifikation gültig ist und alle Endpunkte enthält, die Sie dokumentieren möchten.

    Für Endpunkte, bei denen Sie Metadaten oder Inhalte anpassen möchten, fügen Sie die Erweiterung `x-mint` zum Endpunkt hinzu. Weitere Details finden Sie unter [x-mint extension](/de/api-playground/openapi-setup#x-mint-extension).

    Für Endpunkte, die Sie aus Ihrer Dokumentation ausschließen möchten, fügen Sie die Erweiterung `x-hidden` zum Endpunkt hinzu.

    <Info>
      Validieren Sie Ihre OpenAPI-Datei mit dem [Swagger Editor](https://editor.swagger.io/) oder der [Mint CLI](https://www.npmjs.com/package/mint).
    </Info>
  </Step>

  <Step title="Aktualisieren Sie Ihre Navigationsstruktur.">
    Ersetzen Sie `MDX`-Seitenverweise in Ihrer `docs.json` durch OpenAPI-Endpunkte.

    ```json
    "navigation": {
      "groups": [
        {
          "group": "API Reference",
          "openapi": "/path/to/openapi.json",
          "pages": [
            "overview",
            "authentication",
            "introduction",
            "GET /health",
            "quickstart", 
            "POST /users",
            "GET /users/{id}",
            "advanced-features"
          ]
        }
      ]
    }
    ```
  </Step>

  <Step title="Entfernen Sie alte MDX-Dateien.">
    Nachdem Sie überprüft haben, dass Ihre neue Navigation korrekt funktioniert, entfernen Sie die `MDX`-Endpunktdateien, die Sie nicht mehr benötigen.
  </Step>
</Steps>

<div id="navigation-patterns">
  ## Navigationsmuster
</div>

Sie können steuern, wie Ihre API-Dokumentation in der Navigation erscheint.

<div id="mixed-content-navigation">
  ### Navigation mit gemischten Inhalten
</div>

Kombinieren Sie automatisch generierte API-Seiten mit anderen Seiten:

```json
"navigation": {
  "groups": [
    {
      "group": "API Reference",
      "openapi": "openapi.json",
      "pages": [
        "api/overview",
        "GET /users",
        "POST /users", 
        "api/authentication"
      ]
    }
  ]
}
```

<div id="multiple-api-versions">
  ### Mehrere API-Versionen
</div>

Organisieren Sie verschiedene API-Versionen mit Tabs oder Gruppen:

```json
"navigation": {
  "tabs": [
    {
      "tab": "API v1",
      "openapi": "specs/v1.json"
    },
    {
      "tab": "API v2", 
      "openapi": "specs/v2.json"
    }
  ]
}
```

<div id="when-to-use-individual-mdx-pages">
  ## Wann einzelne `MDX`-Seiten verwendet werden sollten
</div>

Behalten Sie einzelne `MDX`-Seiten bei, wenn Sie Folgendes benötigen:

- Umfangreiche, individuell angepasste Inhalte pro Endpoint, etwa React-Komponenten oder ausführliche Beispiele.
- Einzigartige Seitenlayouts.
- Experimentelle Dokumentationsansätze für bestimmte Endpoints.

Für die meisten Anwendungsfälle bietet die OpenAPI-Navigation bessere Wartbarkeit und mehr Konsistenz.
