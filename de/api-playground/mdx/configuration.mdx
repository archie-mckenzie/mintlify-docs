---
title: "MDX-Setup"
description: "Generieren Sie Dokumentationsseiten für Ihre API-Endpunkte mit `MDX`"
---

Sie können API-Endpunkte manuell in einzelnen `MDX`-Dateien definieren, statt eine OpenAPI-Spezifikation zu verwenden. Diese Methode bietet Flexibilität für individuelle Inhalte, jedoch empfehlen wir für die meisten Projekte, die API-Dokumentation aus einer OpenAPI-Spezifikationsdatei zu generieren, da sie wartungsfreundlicher und funktionsreicher ist. Das Erstellen von `MDX`-Seiten für eine API kann jedoch sinnvoll sein, um kleine APIs zu dokumentieren oder für Prototyping.

Um Seiten für API-Endpunkte mit `MDX` zu generieren, konfigurieren Sie Ihre API-Einstellungen in `docs.json`, erstellen Sie einzelne `MDX`-Dateien für jeden Endpunkt und verwenden Sie Komponenten wie `<ParamFields />`, um Parameter zu definieren. Aus diesen Definitionen generiert Mintlify interaktive API-Playgrounds, Beispielanfragen und Beispielantworten.

<Steps>
  <Step title="Konfigurieren Sie Ihre API">
    Legen Sie in Ihrer `docs.json`-Datei die Basis-URL und die Authentifizierungsmethode fest:

    ```json
     "api": {
      "mdx": {
        "server": "https://mintlify.com/api", // string array for multiple base URLs
        "auth": {
          "method": "key",
          "name": "x-api-key" // options: bearer, basic, key.
        }
      }
    }
    ```

    Wenn Sie den API-Playground ausblenden möchten, verwenden Sie das Feld `display`. Sie müssen keine Authentifizierungsmethode angeben, wenn Sie den Playground ausblenden.

    ```json
    "api": {
      "playground": {
        "display": "none"
      }
    }
    ```

    Eine vollständige Liste der API-Konfigurationen finden Sie unter [Settings](/de/settings#api-configurations).
  </Step>

  <Step title="Erstellen Sie Ihre Endpunktseiten">
    Für jeden API-Endpunkt sollte es eine entsprechende `MDX`-Datei geben. Definieren Sie am Anfang jeder Datei `title` und `api`:

    ```mdx
    ---
    title: 'Create new user'
    api: 'POST https://api.mintlify.com/user'
    ---
    ```

    Sie können Pfadparameter angeben, indem Sie den Parameternamen dem Pfad hinzufügen und in `{}` einschließen:

    ```bash
    https://api.example.com/v1/endpoint/{userId}
    ```

    <Note>
      Wenn Sie ein `server`-Feld in `docs.json` konfiguriert haben, können Sie relative Pfade wie `/v1/endpoint` verwenden.
    </Note>

    Sie können den global definierten Anzeigemodus für den API-Playground seitenweise überschreiben, indem Sie `playground` zum Frontmatter hinzufügen:

    ```mdx
    ---
    title: 'Create new user'
    api: 'POST https://api.mintlify.com/user'
    playground: 'none'
    ---
    ```

    - `playground: 'interactive'` - Zeigt den interaktiven Playground an.
    - `playground: 'simple'` - Zeigt einen kopierbaren Endpunkt ohne Playground an.
    - `playground: 'none'` - Blendet den Playground aus.
  </Step>

  <Step title="Fügen Sie Ihre Endpunkte zur Dokumentation hinzu">
    Fügen Sie Ihre Endpunktseiten der Seitenleiste hinzu, indem Sie die Pfade dem Feld `navigation` in Ihrer `docs.json` hinzufügen. Weitere Informationen zur Strukturierung Ihrer Dokumentation finden Sie unter [Navigation](/de/navigation).
  </Step>
</Steps>

<div id="enabling-authentication">
  ## Authentifizierung aktivieren
</div>

Sie können Ihrer `docs.json` eine Authentifizierungsmethode hinzufügen, um sie global auf allen Seiten zu aktivieren, oder Sie können sie pro Seite festlegen.

Die Authentifizierungsmethode einer Seite überschreibt die globale Methode, wenn beide definiert sind.

<div id="bearer-token">
  ### Bearer-Token
</div>

<CodeGroup>
  ```json docs.json
  "api": {
      "mdx": {
        "auth": {
          "method": "bearer"
        }
      }
  }
  ```

  ```mdx Page Metadata
  ---
  title: "Your page title"
  authMethod: "bearer"
  ---
  ```
</CodeGroup>

<div id="none">
  ### Keine
</div>

Die Authentifizierungsmethode `none` ist nützlich, um nach dem Festlegen eines Standards in der docs.json die Authentifizierung für einen bestimmten Endpunkt zu deaktivieren.

<CodeGroup>
  ```mdx Page Metadata
  ---
  title: "Your page title"
  authMethod: "none"
  ---
  ```
</CodeGroup>
