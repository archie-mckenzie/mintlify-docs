Claude Code ist ein agentenbasiertes Kommandozeilen-Tool, das Sie bei der Pflege Ihrer Dokumentation unterstützt. Es kann neue Inhalte verfassen, bestehende Seiten prüfen und die Doks aktuell halten.

Sie können Claude Code darauf trainieren, Ihre Dokumentationsstandards und Workflows zu verstehen, indem Sie Ihrem Projekt eine `CLAUDE.md`-Datei hinzufügen und diese im Laufe der Zeit verfeinern.

<div id="getting-started">
  ## Erste Schritte
</div>

**Voraussetzungen:**
- Aktives Claude-Abo (Pro, Max oder API-Zugang)

**Einrichtung:**
1. Installiere Claude Code:
  ```bash
  npm install -g @anthropic-ai/claude-code
  ```
2. Navigiere zu deinem Docs-Verzeichnis.
3. (Optional) Füge die untenstehende Datei `CLAUDE.md` zu deinem Projekt hinzu.
4. Starte mit `claude`.

<div id="claudemd-template">
  ## CLAUDE.md-Vorlage
</div>

Speichere eine Datei namens `CLAUDE.md` im Root deines Docs-Verzeichnisses, damit Claude Code dein Projekt besser versteht. Diese Datei schult Claude Code in deinen Dokumentationsstandards, Präferenzen und Workflows. Siehe [Manage Claude's memory](https://docs.anthropic.com/en/docs/claude-code/memory) in den Anthropic-Dokumenten für weitere Informationen.

Kopiere diese Beispielvorlage oder passe sie an die Spezifikationen deiner Docs an:

```mdx
# Mintlify documentation

## Working relationship
- You can push back on ideas—this can lead to better documentation. Cite sources and explain your reasoning when you do so
- ALWAYS ask for clarification rather than making assumptions
- NEVER lie, guess, or make up information

## Project context
- Format: MDX files with YAML frontmatter
- Config: docs.json for navigation, theme, settings
- Components: Mintlify components

## Content strategy
- Document just enough for user success - not too much, not too little
- Prioritize accuracy and usability of information
- Make content evergreen when possible
- Search for existing information before adding new content. Avoid duplication unless it is done for a strategic reason
- Check existing patterns for consistency
- Start by making the smallest reasonable changes

## docs.json

- Refer to the [docs.json schema](https://mintlify.com/docs.json) when building the docs.json file and site navigation

## Frontmatter requirements for pages
- title: Clear, descriptive page title
- description: Concise summary for SEO/navigation

## Writing standards
- Second-person voice ("you")
- Prerequisites at start of procedural content
- Test all code examples before publishing
- Match style and formatting of existing pages
- Include both basic and advanced use cases
- Language tags on all code blocks
- Alt text on all images
- Relative paths for internal links

## Git workflow
- NEVER use --no-verify when committing
- Ask how to handle uncommitted changes before starting
- Create a new branch when no clear branch exists for changes
- Commit frequently throughout development
- NEVER skip or disable pre-commit hooks

## Do not
- Skip frontmatter on any MDX file
- Use absolute URLs for internal links
- Include untested code examples
- Make assumptions - always ask for clarification
```

<div id="sample-prompts">
  ## Beispiel-Prompts
</div>

Sobald Sie Claude Code eingerichtet haben, probieren Sie diese Prompts aus, um zu sehen, wie es bei gängigen Dokumentationsaufgaben hilft. Sie können diese Beispiele direkt kopieren und einfügen oder für Ihre spezifischen Anforderungen anpassen.

<div id="convert-notes-to-polished-docs">
  ### Notizen in ausgereifte Docs umwandeln
</div>

Verwandeln Sie Rohentwürfe in sauber strukturierte Markdown-Seiten mit Komponenten und Frontmatter.

**Beispiel-Prompt:**
```text wrap
Convert this text into a properly formatted MDX page: [paste your text here]
```

<div id="review-docs-for-consistency">
  ### Docs auf Konsistenz prüfen
</div>

Erhalten Sie Vorschläge zur Verbesserung von Stil, Formatierung und Nutzung von Komponenten.

**Beispiel-Prompt:**
```text wrap
Review the files in docs/ and suggest improvements for consistency and clarity
```

<div id="update-docs-when-features-change">
  ### Docs aktualisieren, wenn sich Funktionen ändern
</div>

Halten Sie die Dokumentation aktuell, wenn sich Ihr Produkt weiterentwickelt.

**Beispiel-Prompt:**
```text wrap
Our API now requires a version parameter. Update our docs to include version=2024-01 in all examples
```

<div id="generate-comprehensive-code-examples">
  ### Umfassende Codebeispiele erstellen
</div>

Erstellen Sie mehrsprachige Beispiele mit Fehlerbehandlung.

**Beispiel-Prompt:**
```text wrap
Create code examples for [your API endpoint] in JavaScript, Python, and cURL with error handling
```

<div id="extending-claude-code">
  ## Erweiterung von Claude Code
</div>

Über das reine manuelle Prompten von Claude Code hinaus können Sie es in Ihre bestehenden Workflows integrieren.

<div id="automation-with-github-actions">
  ### Automatisierung mit GitHub Actions
</div>

Führen Sie Claude Code automatisch aus, sobald sich Code ändert, um die Dokumentation aktuell zu halten. Sie können Dokumentationsreviews bei Pull Requests auslösen oder Beispiele aktualisieren, wenn API-Änderungen erkannt werden.

<div id="multi-instance-workflows">
  ### Multi-Instanz-Workflows
</div>

Verwenden Sie separate Claude-Code-Sitzungen für unterschiedliche Aufgaben – eine zum Verfassen neuer Inhalte und eine weitere für Review und Qualitätssicherung. So bleiben Inhalte konsistent, und Probleme, die einer einzelnen Sitzung entgehen könnten, werden eher erkannt.

<div id="team-collaboration">
  ### Teamzusammenarbeit
</div>

Teilen Sie Ihre verfeinerte Datei `CLAUDE.md` mit Ihrem Team, um einheitliche Dokumentationsstandards über alle Beitragenden hinweg sicherzustellen. Teams entwickeln häufig projektspezifische Prompts und Workflows, die dann Teil ihres Dokumentationsprozesses werden.

<div id="custom-commands">
  ### Benutzerdefinierte Befehle
</div>

Erstellen Sie wiederverwendbare Slash-Befehle in `.claude/commands/` für häufig genutzte, dokumentationsbezogene Aufgaben, die spezifisch für Ihr Projekt oder Team sind.
