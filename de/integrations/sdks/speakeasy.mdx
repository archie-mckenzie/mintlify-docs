Automatisch generierte Code-Snippets aus Speakeasy-SDKs lassen sich direkt in die Mintlify-API-Referenz integrieren. SDK-Beispielsnippets werden im [interaktiven API-Playground](/de/api-playground/overview) von Mintlify-basierten Dokumentationsseiten angezeigt.

<Frame>
  ![Ein Mintlify-API-Playground mit Speakeasy-Code-Snippets.](/images/speakeasy/mintlify-with-speakeasy-openapi.png)
</Frame>

<div id="prerequisites">
  ## Voraussetzungen
</div>

Um Mintlify in Speakeasy zu integrieren, benötigen Sie Folgendes:

- Ein [Mintlify-Dokumentationsrepository](/de/quickstart#creating-the-repository).
- Ein von Speakeasy generiertes SDK mit einer konfigurierten [URL für automatisierte Codebeispiele](https://www.speakeasy.com/docs/code-samples/automated-code-sample-urls).

<div id="setting-up-the-integration">
  ## Einrichten der Integration
</div>

Um Speakeasy mit Mintlify zu integrieren, müssen Sie die öffentliche URL der kombinierten Spec der API aus dem Registry abrufen und Ihre `docs.json`-Konfigurationsdatei aktualisieren.

<div id="get-the-apis-combined-spec-public-url-from-the-registry">
  ### Öffentliche URL der kombinierten Spec der API aus dem Registry abrufen
</div>

Navigieren Sie zu Ihrem [Speakeasy-Dashboard](https://app.speakeasy.com) und öffnen Sie den Tab **API Registry**. Öffnen Sie den Eintrag `*-with-code-samples` für die API.

<Frame>
  ![Screenshot der Speakeasy-API-Registry-Seite. Der Tab API Registry ist mit einem roten Quadrat und der Nummer 1 hervorgehoben und der Eintrag für die API ist mit einem roten Quadrat und der Nummer 2 hervorgehoben.](/images/speakeasy/openapi-registry-and-combined-spec.png)
</Frame>

<Note>
  Wenn der Eintrag nicht mit **Combined Spec** gekennzeichnet ist, stellen Sie sicher, dass für die API eine [automatische Codebeispiel-URL](https://www.speakeasy.com/docs/code-samples/automated-code-sample-urls) konfiguriert ist.
</Note>

Kopieren Sie auf der Seite des Registry-Eintrags die bereitgestellte öffentliche URL.

<Frame>
  ![Screenshot, der den Registry-Eintrag der kombinierten Spec zeigt, wobei die Copy-URL-Funktion mit einem roten Quadrat hervorgehoben ist.](/images/speakeasy/copy-combined-spec-url.png)
</Frame>

<div id="update-your-docsjson-configuration-file">
  ### Aktualisieren Sie Ihre `docs.json`-Konfigurationsdatei
</div>

Fügen Sie die URL der kombinierten Spec zu einem Abschnitt **Anchors** oder **Tabs** in Ihrer `docs.json`-Datei hinzu.

Fügen Sie die URL der kombinierten Spec zu einem Anchor hinzu, indem Sie das Feld `anchor` in Ihrer `docs.json`-Datei wie folgt aktualisieren:

```json docs.json
{
  "anchors": [
    {
      "name": "API Reference",
      // !mark
      "openapi": "SPEAKEASY_COMBINED_SPEC_URL",
      "url": "api-reference",
      "icon": "square-terminal"
    }
  ]
}
```

Fügen Sie die URL der kombinierten Spec zu einem Tab hinzu, indem Sie das Feld `tab` in der Datei `docs.json` wie folgt aktualisieren:

```json docs.json
{
  "tabs": [
    {
      "name": "API Reference",
      "url": "api-reference",
      // !mark
      "openapi": "SPEAKEASY_COMBINED_SPEC_URL"
    }
  ]
}
```

Von Speakeasy generierte Codebeispiele können nun in Ihrer API-Dokumentation angezeigt und im API-Playground verwendet werden.
