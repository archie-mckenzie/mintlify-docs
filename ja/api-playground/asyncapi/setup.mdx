<div id="add-an-asyncapi-specification-file">
  ## AsyncAPI 仕様ファイルを追加する
</div>

WebSocket のページ作成を始めるにあたっては、[AsyncAPI specification](https://www.asyncapi.com/docs/reference/specification/v3.0.0) に準拠した有効な AsyncAPI スキーマドキュメントを、JSON または YAML 形式で用意してください。スキーマは AsyncAPI 仕様 3.0 以上に準拠している必要があります。

<Tip>
  AsyncAPI スキーマの有効性を確認するには、
  [AsyncAPI Studio](https://studio.asyncapi.com/) に貼り付けて検証できます
</Tip>

<div id="auto-populate-websockets-pages">
  ## Websocket ページの自動生成
</div>

`docs.json` のナビゲーション内の任意のタブまたはグループに `asyncapi` フィールドを追加できます。このフィールドには、ドキュメントリポジトリ内の AsyncAPI スキーマドキュメントへのパス、ホストされた AsyncAPI スキーマドキュメントの URL、または AsyncAPI スキーマドキュメントへのリンクの配列を指定できます。Mintlify は各 AsyncAPI の WebSocket チャンネルごとにページを自動生成します。

**タブの例:**

<CodeGroup>

```json Local File {5}
"navigation": {
  "tabs": [
    {
        "tab": "API Reference",
        "asyncapi": "/path/to/asyncapi.json"
    }
  ]
}

```

```json Remote URL {5}
"navigation": {
  "tabs": [
    {
        "tab": "API Reference",
        "asyncapi": "https://github.com/asyncapi/spec/blob/master/examples/simple-asyncapi.yml"
    }
  ]
}
```

</CodeGroup>

**グループの例:**

```json {8-11}
"navigation": {
  "tabs": [
    {
      "tab": "AsyncAPI",
      "groups": [
        {
          "group": "Websockets",
          "asyncapi": {
            "source": "/path/to/asyncapi.json",
            "directory": "api-reference"
          }
        }
      ]
    }
  ]
}
```

<Note>
  directory フィールドは任意です。指定がない場合、ファイルはドキュメントリポジトリの
  **api-reference** フォルダーに配置されます。
</Note>

<div id="channel-page">
  ## チャネルページ
</div>

チャネルの並び順をより柔軟に制御したい場合や、特定のチャネルだけを参照したい場合は、フロントマターに `asyncapi` フィールドを含む MDX ファイルを作成します。

```mdx
---
title: "Websocket Channel"
asyncapi: "/path/to/asyncapi.json channelName"
---
```
