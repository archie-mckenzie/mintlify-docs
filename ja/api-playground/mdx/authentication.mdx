<div id="enabling-authentication">
  ## 認証の有効化
</div>

`docs.json` に認証方式を追加して全ページで有効化することも、ページごとに設定することもできます。

ページの認証方式は、両方が設定されている場合、グローバル設定を上書きします。

<div id="bearer-token">
  ### Bearer トークン
</div>

<CodeGroup>

```json docs.json
"api": {
    "mdx": {
      "auth": {
        "method": "bearer"
      }
    }
}
```

```mdx Page Metadata
---
title: "ページのタイトル"
authMethod: "bearer"
---
```

</CodeGroup>

<div id="basic-authentication">
  ### Basic 認証
</div>

<CodeGroup>

```json docs.json
"api": {
    "mdx": {
      "auth": {
        "method": "basic"
      }
    }
}
```

```mdx Page Metadata
---
title: "ページタイトル"
authMethod: "basic"
---
```

</CodeGroup>

<div id="api-key">
  ### API キー
</div>

<CodeGroup>

```json docs.json
"api": {
    "mdx": {
      "auth": {
        "method": "key",
        "name": "x-api-key"
      }
    }
}
```

```mdx Page Metadata
---
title: "ページタイトル"
authMethod: "key"
---
```

</CodeGroup>

<div id="none">
  ### None
</div>

"none" 認証方式は、docs.json でデフォルトを設定した後に、特定のエンドポイントだけ認証を無効化する際に便利です。

<CodeGroup>
```mdx Page Metadata
---
title: "Your page title"
authMethod: "none"
---
```
</CodeGroup>
