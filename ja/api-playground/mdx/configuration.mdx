---
title: 'MDX のセットアップ'
description: '`MDX` を使用して API エンドポイント向けのドキュメントページを生成する'
---

`OpenAPI` 仕様を使わずに、個々の `MDX` ファイルで API エンドポイントを手動で定義できます。この方法はカスタムコンテンツに柔軟ですが、保守性と機能性の観点から、ほとんどのプロジェクトでは `OpenAPI` 仕様ファイルから API ドキュメントを生成することを推奨します。ただし、小規模な API のドキュメント化やプロトタイピングには `MDX` ページの作成が有用です。

`MDX` を使用して API エンドポイントのページを生成するには、`docs.json` で API 設定を構成し、各エンドポイント用に個別の `MDX` ファイルを作成し、`<ParamFields />` などのコンポーネントでパラメータを定義します。これらの定義から、Mintlify はインタラクティブな API プレイグラウンド、リクエスト例、レスポンス例を生成します。

<Steps>
  <Step title="API を構成する">
    `docs.json` ファイルで、ベース URL と認証方式を定義します:

    ```json
     "api": {
      "mdx": {
        "server": "https://mintlify.com/api", // 複数のベース URL の場合は string 配列
        "auth": {
          "method": "key",
          "name": "x-api-key" // オプション: bearer, basic, key
        }
      }
    }
    ```

    API プレイグラウンドを非表示にしたい場合は、`display` フィールドを使用します。プレイグラウンドを非表示にする場合は認証方式を含める必要はありません。

    ```json
    "api": {
      "playground": {
        "display": "none"
      }
    }
    ```

    API 設定の一覧は [Settings](/ja/settings#api-configurations) を参照してください。
  </Step>

  <Step title="エンドポイントページを作成する">
    各 API エンドポイントページには対応する `MDX` ファイルが必要です。各ファイルの先頭で、`title` と `api` を定義します:

    ```mdx
    ---
    title: 'Create new user'
    api: 'POST https://api.mintlify.com/user'
    ---
    ```

    パスパラメータは、パスにパラメータ名を `{}` で囲んで追加することで指定できます:

    ```bash
    https://api.example.com/v1/endpoint/{userId}
    ```

    <Note>
      `docs.json` に `server` フィールドが設定されている場合は、`/v1/endpoint` のように相対パスを使用できます。
    </Note>

    ページ単位で API プレイグラウンドのグローバル表示モードを上書きするには、フロントマターに `playground` を追加します:

    ```mdx
    ---
    title: 'Create new user'
    api: 'POST https://api.mintlify.com/user'
    playground: 'none'
    ---
    ```

    * `playground: 'interactive'` - インタラクティブなプレイグラウンドを表示。
    * `playground: 'simple'` - プレイグラウンドなしのコピー可能なエンドポイントを表示。
    * `playground: 'none'` - プレイグラウンドを非表示。
  </Step>

  <Step title="エンドポイントをドキュメントに追加する">
    `docs.json` の `navigation` フィールドにパスを追加して、エンドポイントページをサイドバーに追加します。ドキュメントの構成については [Navigation](/ja/navigation) を参照してください。
  </Step>
</Steps>

<div id="enabling-authentication">
  ## 認証を有効化する
</div>

`docs.json` に認証方法を追加して全ページでグローバルに有効化することも、ページ単位で設定することもできます。

ページの認証方法は、両方が設定されている場合、グローバル設定よりも優先されます。

<div id="bearer-token">
  ### ベアラートークン
</div>

<CodeGroup>
  ```json docs.json
  "api": {
      "mdx": {
        "auth": {
          "method": "bearer"
        }
      }
  }
  ```

  ```mdx Page Metadata
  ---
  title: "ページタイトル"
  authMethod: "bearer"
  ---
  ```
</CodeGroup>

<div id="basic-authentication">
  ### ベーシック認証
</div>

<CodeGroup>
  ```json docs.json
  "api": {
      "mdx": {
        "auth": {
          "method": "basic"
        }
      }
  }
  ```

  ```mdx Page Metadata
  ---
  title: "ページのタイトル"
  authMethod: "basic"
  ---
  ```
</CodeGroup>

<div id="api-key">
  ### API キー
</div>

<CodeGroup>
  ```json docs.json
  "api": {
      "mdx": {
        "auth": {
          "method": "key",
          "name": "x-api-key"
        }
      }
  }
  ```

  ```mdx Page Metadata
  ---

  ```
</CodeGroup>

<div id="none">
  ### None
</div>

`none` 認証方式は、docs.json でデフォルトを設定した後に、特定のエンドポイントで認証を無効化するために便利です。

<CodeGroup>
  ```mdx Page Metadata
  ---
  title: "Your page title"
  authMethod: "none"
  ---
  ```
</CodeGroup>
