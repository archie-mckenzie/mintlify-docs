接続済みのリポジトリに変更をプッシュすると、ドキュメントサイトは自動的にデプロイされます。これには、Mintlify の GitHub アプリが正しくインストールされ、接続されている必要があります。

最新の変更が本番サイトに反映されない場合は、まず docs リポジトリの所有者であるアカウントまたは組織に GitHub アプリがインストールされているか確認してください。詳しくは [GitHub のトラブルシューティング](/ja/settings/github#troubleshooting) を参照してください。

GitHub アプリが接続されているのに変更がデプロイされない場合は、ダッシュボードから手動で再ビルドを実行できます。

<div id="manually-triggering-a-deployment">
  ## デプロイを手動で実行する
</div>

<Steps>
  <Step title="最新のコミットが成功していることを確認する。">
    最新のコミットがドキュメントリポジトリに反映され、エラーが発生していないことを確認します。
  </Step>

  <Step title="デプロイを手動で実行する。">
    [ダッシュボード](https://dashboard.mintlify.com)にアクセスし、Deploy ボタンを選択します。

    </Frame>
  </Step>
</Steps>
