title: "非表示ページ"
description: "ナビゲーションからページを除外する"

---

非表示ページはサイトのナビゲーションには表示されませんが、URL を知っていれば誰でも公開アクセスできます。

非表示ページは、サイト上でアクセス可能にしておきたい、または AI ツールのコンテキストとして参照させたいが、ナビゲーションからは見つからないようにしたいコンテンツに使用します。

厳格なアクセス制御が必要なコンテンツには、[authentication](/ja/authentication-personalization/authentication-setup) を設定してください。

特定のユーザーグループに対してページを非表示にしたい場合は、パーソナライゼーションで[ページの可視性](/ja/authentication-personalization/overview#page-visibility)を制御してください。

<div id="hiding-a-page">
  ## ページを非表示にする
</div>

`docs.json` のナビゲーションに含まれていない場合、ページは非表示になります。ページを非表示にするには、ナビゲーション構造から削除してください。

非表示のページでも、ファイルパスに基づく通常のページと同じ URL 構造を使用します。たとえば、`guides/hidden-page.mdx` は `docs.yoursite.com/guides/hidden-page` でアクセスできます。

[非表示ページの例](/ja/guides/hidden-page-example)をご覧ください。

<Note>
  サイドバー、ドロップダウン、タブなどの一部のナビゲーション要素は、非表示ページで空になったり、レイアウトが崩れたりする場合があります。
</Note>

<div id="hiding-a-group-of-pages">
  ## ページグループを非表示にする
</div>

docs.json ファイルで `hidden` プロパティを `true` に設定すると、ページグループは非表示になります:

```json highlight={4}
"groups": [
  {
    "group": "Getting started",
    "hidden": true,
    "pages": [
      "index",
      "quickstart"
    ]
  },
  {
    "group": "Guides",
    "pages": [
      "guides/hidden-page.mdx",
      "guides/hidden-groups.mdx"
    ]
  }
]
```

この例では、`Getting started` グループは非表示ですが、`Guides` グループは表示されます。

<div id="hiding-a-tab">
  ### タブを非表示にする
</div>

`docs.json` ファイルに `hidden` プロパティを追加すると、タブを非表示にできます：

```json highlight={4}
"tabs": [
  {
    "tab": "Home",
    "hidden": true,
    "pages": [
      "index",
      "quickstart"
    ]
  }
]
```

<div id="search-seo-and-ai-indexing">
  ## 検索、SEO、AIのインデックス化
</div>

デフォルトでは、非表示ページは検索エンジン、ドキュメント内検索、そしてAI Assistantのコンテキストとしてのインデックス化から除外されます。非表示ページを検索結果およびAssistantのコンテキストに含めるには、`docs.json` に `seo` プロパティを追加します:

```json
"seo": {
    "indexing": "all"
}
```

特定のページを除外するには、そのフロントマターに `noindex: "true"` を追加します。
