Claude Code は、ドキュメントの保守に役立つエージェント型のコマンドラインツールです。新しいコンテンツの作成、既存ページのレビュー、ドキュメントの最新化を行えます。

`CLAUDE.md` ファイルをプロジェクトに追加し、継続的に磨き込むことで、Claude Code にドキュメントの標準やワークフローを理解させられます。

<div id="getting-started">
  ## はじめに
</div>

**前提条件:**

- Claude の有効なサブスクリプション（Pro、Max、または API アクセス）

**セットアップ:**

1. Claude Code をインストールします:

```bash
  npm install -g @anthropic-ai/claude-code
  ```

2. docs ディレクトリに移動します。
3. （任意）以下の `CLAUDE.md` ファイルをプロジェクトに追加します。
4. `claude` を実行して開始します。

<div id="claudemd-template">
  ## CLAUDE.md テンプレート
</div>

`CLAUDE.md` ファイルをドキュメントディレクトリのルートに保存して、Claude Code がプロジェクトを理解できるようにします。このファイルは、ドキュメントの標準、好み、ワークフローについて Claude Code を学習させます。詳しくは Anthropic のドキュメントにある [Manage Claude's memory](https://docs.anthropic.com/en/docs/claude-code/memory) を参照してください。

以下のサンプルテンプレートをコピーするか、ドキュメント仕様に合わせて調整してください:

```mdx
# Mintlify documentation

## Working relationship
- アイデアに異議を唱えても構いません—その方がより良いドキュメントにつながることがあります。出典を示し、その理由を説明してください
- 推測せず、必ず不明点は確認すること
- 決して嘘をつかない・推測で埋めない・情報をでっち上げないこと

## Project context
- Format: YAML フロントマター付きの MDX ファイル
- Config: ナビゲーション、テーマ、設定用の docs.json
- Components: Mintlify コンポーネント

## Content strategy
- ユーザーが成功するのに必要十分な情報だけを文書化する（過不足なく）
- 情報の正確性と使いやすさを最優先する
- 可能な限りコンテンツを長期的に有用（エバーグリーン）に保つ
- 新規追加の前に既存情報を検索する。戦略的な理由がない限り重複は避ける
- 既存のパターンに一貫性があるか確認する
- まずは最小限で妥当な変更から始める

## docs.json

- docs.json ファイルおよびサイトナビゲーションを作成する際は、[docs.json スキーマ](https://mintlify.com/docs.json) を参照すること

## Frontmatter requirements for pages
- title: 明確で説明的なページタイトル
- description: SEO/ナビゲーションのための簡潔な要約

## Writing standards
- 二人称（"you"）で記述する
- 手順コンテンツの冒頭に前提条件を記載する
- 公開前にすべてのコード例をテストする
- 既存ページの文体と書式に合わせる
- 基本から高度なユースケースまで含める
- すべてのコードブロックに言語タグを付ける
- すべての画像に代替テキスト（alt）を付ける
- 内部リンクには相対パスを使用する

## Git workflow
- コミット時に --no-verify を絶対に使用しない
- 着手前に、未コミット変更の扱いを確認する
- 変更に適切なブランチがなければ新規ブランチを作成する
- 開発中はこまめにコミットする
- pre-commit フックを無効化・スキップしないこと

## Do not
- いかなる MDX ファイルでもフロントマターを省略しない
- 内部リンクに絶対 URL を使用しない
- テストしていないコード例を含めない
- 推測しない—必ず不明点は確認すること
```

<div id="sample-prompts">
  ## サンプルプロンプト
</div>

Claude Code のセットアップが完了したら、次のプロンプトを試して、一般的なドキュメント作成タスクにどう役立つか確認してください。これらの例はそのままコピー＆ペーストすることも、用途に合わせて調整することもできます。

<div id="convert-notes-to-polished-docs">
  ### メモを完成度の高いドキュメントに変換
</div>

ラフな下書きを、コンポーネントとフロントマターを備えた適切なMarkdownページに仕上げます。

**プロンプト例:**

```text wrap
Convert this text into a properly formatted MDX page: [paste your text here]
```

<div id="review-docs-for-consistency">
  ### ドキュメントの一貫性をレビュー
</div>

スタイル、書式、コンポーネントの使用方法の改善提案を受け取れます。

**プロンプト例:**

```text wrap
docs/ 内のファイルをレビューして、一貫性と明確性を高めるための改善点を提案してください
```

<div id="update-docs-when-features-change">
  ### 機能変更時にドキュメントを更新する
</div>

プロダクトの進化に合わせてドキュメントを常に最新に保ちましょう。

**プロンプト例:**

```text wrap
Our API now requires a version parameter. Update our docs to include version=2024-01 in all examples
```

<div id="generate-comprehensive-code-examples">
  ### 網羅的なコード例を生成する
</div>

エラーハンドリング付きの多言語コード例を作成します。

**プロンプト例:**

```text wrap
Create code examples for [your API endpoint] in JavaScript, Python, and cURL with error handling
```

<div id="extending-claude-code">
  ## Claude Code を拡張する
</div>

手動で Claude Code にプロンプトを送る以外にも、既存のワークフローに統合できます。

<div id="automation-with-github-actions">
  ### GitHub Actions による自動化
</div>

コード変更に合わせて Claude Code を自動実行し、ドキュメントを最新に保ちます。プルリクエストでドキュメントレビューをトリガーしたり、API の変更を検知した際にサンプルを更新したりできます。

<div id="multi-instance-workflows">
  ### マルチインスタンスのワークフロー
</div>

タスクごとにClaude Codeのセッションを分けて使いましょう。例えば、新規コンテンツの作成用と、レビューや品質保証用でセッションを分けます。こうすることで一貫性を保ち、単一のセッションでは見落としがちな問題を洗い出せます。

<div id="team-collaboration">
  ### チームでのコラボレーション
</div>

改善した `CLAUDE.md` ファイルをチームと共有し、すべての貢献者で一貫したドキュメント標準を維持しましょう。チームでは、プロジェクト特有のプロンプトやワークフローが整備され、ドキュメント作成プロセスの一部になることがよくあります。

<div id="custom-commands">
  ### カスタムコマンド
</div>

プロジェクトやチーム固有のよく使うドキュメント作業向けに、再利用可能なスラッシュコマンドを `.claude/commands/` に作成します。
