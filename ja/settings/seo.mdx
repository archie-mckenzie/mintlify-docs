Mintlify は、多くの SEO のベストプラクティスを自動的に処理します。例えば次のとおりです。

* メタタグの生成
* サイトマップと `robots.txt` の生成
* セマンティックな HTML 構造
* モバイル最適化

`docs.json` または各ページのフロントマターに `metatags` フィールドを追加することで、サイトのメタタグを完全にカスタマイズできます。

<div id="global-meta-tags">
  ## グローバルメタタグ
</div>

`docs.json` に `metadata` フィールドを追加して、デフォルトのメタタグを設定できます。

```json
"seo": {
    "metatags": {
        "og:image": "デフォルトのメタタグ用画像へのリンク"
    }
}
```

<div id="page-specific-meta-tags">
  ## ページ固有のメタタグ
</div>

メタタグは次のようにページのメタデータに追加します：

```mdx
---
title: 'Your Example Page Title'
'og:image': 'link to your meta tag image'
---
```

<Warning>
  コロンを含むメタタグは、ページのメタデータに記述する際、タグ名を引用符で囲む必要があります。
</Warning>

<div id="sitemaps-and-robotstxt-files">
  ## サイトマップと robots.txt ファイル
</div>

Mintlify は `sitemap.xml` ファイルと `robots.txt` ファイルを自動生成します。ドキュメントサイトの URL に `/sitemap.xml` を付けると、サイトマップを表示できます。

既定では、`docs.json` に含まれるページのみが対象です。非表示のリンクも含めるには、`docs.json` に `seo.indexing` を追加します:

```json
"seo": {
    "indexing": all
}
```

<div id="custom-sitemaps-and-robotstxt-files">
  ### カスタムの sitemap と robots.txt ファイル
</div>

カスタムの `sitemap.xml` または `robots.txt` を追加するには、プロジェクトのルートに `sitemap.xml` または `robots.txt` を作成します。カスタムファイルを追加すると、同名の自動生成ファイルが上書きされます。カスタムファイルを削除すると、既定のファイルが再び適用されます。

<div id="disabling-indexing">
  ## インデックスの無効化
</div>

検索エンジンによるページのインデックス化を防ぎたい場合は、ページの[frontmatter](ja/pages)に次を追加します:

```
---
noindex: true
---
```

`docs.json`で`metadata.robots`フィールドを`"noindex"`に設定すると、ドキュメント内のすべてのページに対して`noindex`を指定できます:

```json
"seo": {
    "metatags": {
      "robots": "noindex"
    }
  }
```

<div id="seo-best-practices">
  ## SEO のベストプラクティス
</div>

<AccordionGroup>
  <Accordion title="わかりやすいタイトルと説明文を書く">
    * 明確で説明的なページタイトルを使う（50〜60文字）
    * 魅力的な説明文を書く（150〜160文字）
    * 関連キーワードを含める
    * 各ページでタイトルと説明文を固有にする
  </Accordion>

  <Accordion title="コンテンツ構造を最適化する">
    * 適切な見出し階層を使う（H1 → H2 → H3）
    * まず人に向けて書き、検索エンジンは次に考える
    * 見出しと本文に関連キーワードを含める
    * URL は短く、説明的にし、階層構造で整理する
    * 長文は小見出しやリストで分割する
  </Accordion>

  <Accordion title="内部リンクの戦略">
    * ドキュメント内で関連ページにリンクする
    * 「click here」ではなく説明的なアンカーテキストを使う
    * 関連する概念を相互にリンクしてトピッククラスターを作る
    * 自動クロスリファレンス機能を活用する
  </Accordion>

  <Accordion title="画像の SEO">
    * 画像には説明的なファイル名を付ける
    * アクセシビリティと SEO のために必ず alt テキストを入れる
    * 高速表示のために画像ファイルサイズを最適化する
    * コンテンツを支える適切な画像を使う
  </Accordion>
</AccordionGroup>
