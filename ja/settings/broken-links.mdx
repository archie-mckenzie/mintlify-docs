docsフォルダ内のファイルのパスを変更すると、そのページのURLパスも変更されます。これは、ドキュメント構成を見直したり、サイドバーのタイトルを変更したりする際に発生することがあります。

<div id="broken-links">
  ## 壊れたリンク
</div>

CLIで壊れたリンクを検出します。[CLI をインストール](/ja/installation)して、次のコマンドを実行します:

```bash
mint broken-links
```

CLI は、ドキュメント内に存在しない相対リンクを特定します。

<div id="redirects">
  ## リダイレクト
</div>

`docs.json` ファイルに `redirects` フィールドを追加して、301 リダイレクトを設定します。

```json
"redirects": [
  {
    "source": "/source/path",
    "destination": "/destination/path"
  }
]
```

これにより、`/source/path` は恒久的に `/destination/path` へリダイレクトされ、元のページの既存の SEO が失われません。

ワイルドカードのパスに一致させるには、パラメータの後に `*` を付けます。次の例では、`/beta/:slug*` は `/beta/introduction` に一致し、`/v2/introduction` にリダイレクトされます。

```json
"redirects": [
  {
    "source": "/beta/:slug*",
    "destination": "/v2/:slug*"
  }
]
```
