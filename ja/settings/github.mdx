MintlifyはGitHubアプリを使用して、ドキュメントをGitHubリポジトリと自動的に同期します。

<div id="installing-the-github-app">
  ## GitHubアプリのインストール
</div>

ドキュメント用リポジトリを所有するGitHubアカウントまたは組織にMintlify GitHub Appをインストールします。インストールには、組織の所有者権限またはリポジトリの管理者権限が必要です。必要な権限がない場合は、リポジトリ所有者による承認が必要です。

[ダッシュボード](https://dashboard.mintlify.com/settings/organization/github-app)からMintlify GitHub Appをインストールします。

<Tip>
ドキュメントをホストしているリポジトリのみにアクセス権を付与することを推奨します。
</Tip>

<div id="permissions">
  ## 権限
</div>

GitHubアプリをインストールすると、次の権限の付与を求められます。

読み取り権限:

* `metadata`: リポジトリの基本情報

読み取りおよび書き込み権限:

* `checks`: プルリクエストにステータスチェックを作成
* `code`: docsブランチへのコミット時のファイル変更を読み取り
* `deployments`: プルリクエスト用のプレビュー環境を生成
* `pull requests`: Webエディターからブランチおよびプルリクエストを作成
* `workflows`: ドキュメントのデプロイ用のGitHub Actionsを管理

<Info>
このアプリは、明示的にアクセスを許可したリポジトリにのみアクセスします。ブランチ保護ルールが有効な場合、アプリは保護されたブランチに直接プッシュできません。
</Info>

<div id="managing-repository-access">
  ## リポジトリへのアクセス管理
</div>

GitHub App をインストールする際、すべてのリポジトリか特定のリポジトリへのアクセス権を付与できます。ドキュメントがあるリポジトリにのみ付与することを推奨します。この設定はいつでも [GitHub App の設定](https://github.com/apps/mintlify/installations/new) で変更できます。

<div id="configuring-docs-source">
  ## ドキュメントソースの設定
</div>

ダッシュボードの [Git Settings](https://dashboard.mintlify.com/settings/deployment/git-settings) セクションで、ドキュメントのビルド元となる組織・リポジトリ・ブランチを変更できます。

<div id="troubleshooting">
  ## トラブルシューティング
</div>

GitHub アプリで問題が発生した場合は、接続をリセットすると多くの問題が解決します。

<Steps>
  <Step title="GitHub から Mintlify アプリをアンインストールする">
    1. GitHub の [installations](https://github.com/settings/installations) にアクセスし、Mintlify アプリの **Configure** を選択します。下部までスクロールして **Uninstall** を選択します。
    2. [Authorized GitHub Apps](https://github.com/settings/apps/authorizations) にアクセスし、Mintlify アプリの **Revoke** を選択します。
  </Step>
  <Step title="Mintlify アプリを再インストールする">
    1. Mintlify ダッシュボードの [Git Settings](https://dashboard.mintlify.com/settings/deployment/git-settings) に移動し、GitHub アプリをインストールします。
    2. ダッシュボードの [My Profile](https://dashboard.mintlify.com/settings/account) でアカウントを認可します。
  </Step>
</Steps>
