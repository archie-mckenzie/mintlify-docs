`docs.json` の [navigation](ja/settings#param-navigation) プロパティは、ドキュメントの構造と情報階層を制御します。

適切にナビゲーションを設定すると、ユーザーが求める情報を正確に見つけられるようにコンテンツを整理できます。

<div id="pages">
  ## Pages
</div>

Pagesは最も基本的なナビゲーションコンポーネントです。Pagesは、ドキュメントを構成するMDXファイルに対応します。

<img className="block dark:hidden pointer-events-none" src="/images/navigation/pages-light.png" alt="" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/pages-dark.png" alt="" />

`navigation` オブジェクトでは、`pages` は配列で、各エントリは[ページファイル](ja/pages)へのパスを参照する必要があります。

```json
{
  "navigation": {
    "pages": [
      "settings",
      "pages",
      "navigation",
      "themes",
      "custom-domain"
    ]
  }
}
```

<div id="groups">
  ## グループ
</div>

サイドバーのナビゲーションをセクションごとに整理するには、グループを使います。グループは入れ子にでき、タグでラベル付けしたり、アイコンで装飾できます。

<img className="block dark:hidden pointer-events-none" src="/images/navigation/groups-light.png" alt="" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/groups-dark.png" alt="" />

`navigation` オブジェクトでは、`groups` は各要素が `group` フィールドと `pages` フィールドを必須とするオブジェクトの配列です。`icon` と `tag` フィールドは任意です。

```json
{
  "navigation": {
    "groups": [
      {
        "group": "Getting started",
        "icon": "play",
        "pages": [
          "quickstart",
          {
            "group": "Editing",
            "icon": "pencil",
            "pages": [
              "installation",
              "editor"
            ]
          }
        ]
      },
      {
        "group": "Writing content",
        "icon": "notebook-text",
        "tag": "NEW",
        "pages": [
          "writing-content/page",
          "writing-content/text"
        ]
      }
    ]
  }
}
```

<div id="tabs">
  ## タブ
</div>

タブは、ドキュメント内に独立したURLパスを持つ明確なセクションを作成します。ドキュメント上部に水平ナビゲーションバーを表示し、ユーザーがセクション間を切り替えられるようにします。

<img className="block dark:hidden pointer-events-none" src="/images/navigation/tabs-light.png" alt="" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/tabs-dark.png" alt="" />

`navigation` オブジェクトでは、`tabs` は配列で、各要素は必須の `tab` フィールドを持つオブジェクトです。さらに、グループ、ページ、アイコン、外部ページへのリンクなど、他のナビゲーション用フィールドを含めることができます。

```json
{
  "navigation": {
    "tabs": [
      {
        "tab": "API reference",
        "icon": "square-terminal",
        "pages": [
          "api-reference/get",
          "api-reference/post",
          "api-reference/delete"
        ]
      },
      {
        "tab": "SDKs",
        "icon": "code",
        "pages": [
          "sdk/fetch",
          "sdk/create",
          "sdk/delete"
        ]
      },
      {
        "tab": "Blog",
        "icon": "newspaper",
        "href": "https://external-link.com/blog"
      }
    ]
  }
}
```

<div id="menus">
  ### メニュー
</div>

メニューはタブにドロップダウン式のナビゲーション項目を追加します。メニューを使うと、ユーザーはタブ内の特定のページへ直接移動できます。

`navigation` オブジェクトでは、`menu` は配列で、各要素は必須の `item` フィールドを持つオブジェクトです。さらに、グループ、ページ、アイコン、外部ページへのリンクなどの他のナビゲーション用フィールドを含められます。

```json
{
  "navigation": {
    "tabs": [
      {
        "tab": "Developer tools",
        "icon": "square-terminal",
        "menu": [
          {
            "item": "API reference",
            "icon": "rocket",
            "groups": [
              {
                "group": "Core endpoints",
                "icon": "square-terminal",
                "pages": [
                  "api-reference/get",
                  "api-reference/post",
                  "api-reference/delete"
                ]
              }
            ]
          },
          {
            "item": "SDKs",
            "icon": "code",
            "description": "SDKs are used to interact with the API.",
            "pages": [
              "sdk/fetch",
              "sdk/create",
              "sdk/delete"
            ]
          }
        ]
      }
    ]
  }
}
```

<div id="anchors">
  ## アンカー
</div>

アンカーは、サイドバー上部に常設のナビゲーション項目を追加します。コンテンツを区分したり、外部リソースへのクイックアクセスを提供したり、目立つCTAを用意するためにアンカーを活用できます。

<img className="block dark:hidden pointer-events-none" src="/images/navigation/anchors-light.png" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/anchors-dark.png" />

`navigation` オブジェクトでは、`anchors` は配列で、各要素は必須の `anchor` フィールドを持つオブジェクトです。加えて、グループ、ページ、アイコン、外部ページへのリンクなど、他のナビゲーション用フィールドを含められます。

```json
{
  "navigation": {
    "anchors": [
      {
        "anchor": "Documentation",
        "icon": "book-open",
        "pages": [
          "quickstart",
          "development",
          "navigation"
        ]
      },
      {
        "anchor": "API reference",
        "icon": "square-terminal",
        "pages": [
          "api-reference/get",
          "api-reference/post",
          "api-reference/delete"
        ]
      },
      {
        "anchor": "Blog",
        "href": "https://external-link.com/blog"
      }
    ]
  }
}
```

外部リンクのみに遷移するアンカーには、`global` キーワードを使用します。`global` オブジェクト内のアンカーは `href` フィールドが必須で、相対パスを指すことはできません。

グローバルアンカーは、ドキュメントの一部ではないものの、ブログやサポートポータルのようにユーザーがすぐアクセスできるべきリソースへのリンクに特に有用です。

```json
{
  "navigation": {
    "global":  {
      "anchors": [
        {
          "anchor": "Community",
          "icon": "house",
          "href": "https://slack.com"
        },
        {
          "anchor": "Blog",
          "icon": "pencil",
          "href": "https://mintlify.com/blog"
        }
      ]
    },
    "tabs": /*...*/
  }
}
```

<div id="dropdowns">
  ## ドロップダウン
</div>

ドロップダウンは、サイドバーのナビゲーション上部にある展開メニューに含まれます。ドロップダウン内の各項目は、ドキュメント内のセクションへ移動します。

<img className="block dark:hidden pointer-events-none" src="/images/navigation/dropdowns-light.png" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/dropdowns-dark.png" />

`navigation` オブジェクトでは、`dropdowns` は配列で、各要素は必須の `dropdown` フィールドを持ち、グループ、ページ、アイコン、外部ページへのリンクなどの他のナビゲーションフィールドも含められます。

```json
{
  "navigation": {
    "dropdowns": [
      {
        "dropdown": "Documentation",
        "icon": "book-open",
        "pages": [
          "quickstart",
          "development",
          "navigation"
        ]
      },
      {
        "dropdown": "API reference",
        "icon": "square-terminal",
        "pages": [
          "api-reference/get",
          "api-reference/post",
          "api-reference/delete"
        ]
      },
      {
        "dropdown": "Blog",
        "href": "https://external-link.com/blog"
      }
    ]
  }
}
```

<div id="openapi">
  ## OpenAPI
</div>

OpenAPI 仕様をナビゲーション構造に直接組み込み、API ドキュメントを自動生成します。専用の API セクションを作成することも、エンドポイントページを他のナビゲーション要素内に配置することもできます。

ナビゲーション階層の任意のレベルで既定の OpenAPI 仕様を設定できます。子要素は独自の仕様を定義しない限り、その仕様を継承します。

```json
{
  "navigation": {
    "groups": [
      {
        "group": "API reference",
        "openapi": "/path/to/openapi-v1.json",
        "pages": [
          "overview",
          "authentication",
          "GET /users",
          "POST /users",
          {
            "group": "Products",
            "openapi": "/path/to/openapi-v2.json",
            "pages": [
              "GET /products",
              "POST /products"
            ]
          }
        ]
      }
    ]
  }
}
```

ドキュメントでの OpenAPI エンドポイントの参照方法の詳細は、[OpenAPI setup](/ja/api-playground/openapi-setup)を参照してください。

<div id="versions">
  ## バージョン
</div>

ナビゲーションを複数のバージョンに分割できます。バージョンはドロップダウンメニューから選択します。

<img className="block dark:hidden pointer-events-none" src="/images/navigation/versions-light.png" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/versions-dark.png" />

`navigation` オブジェクトでは、`versions` は配列で、各要素は `version` フィールドが必須で、ほかの任意のナビゲーション用フィールドを含められます。

```json
{
  "navigation": {
    "versions": [
      {
        "version": "1.0.0",
        "groups": [
          {
            "group": "Getting started",
            "pages": ["v1/overview", "v1/quickstart", "v1/development"]
          }
        ]
      },
      {
        "version": "2.0.0",
        "groups": [
          {
            "group": "Getting started",
            "pages": ["v2/overview", "v2/quickstart", "v2/development"]
          }
        ]
      }
    ]
  }
}
```

<div id="breadcrumbs">
  ## パンくずリスト
</div>

パンくずリストは、ページ上部にナビゲーションパス全体を表示します。テーマによってはデフォルトで有効な場合もあれば、無効な場合もあります。サイトでパンくずリストを有効にするかどうかは、`docs.json` の `styling` プロパティで制御できます。

<CodeGroup>
  ```json Display full breadcrumbs
  "styling": {
    "eyebrows": "breadcrumbs"
  }
  ```

  ```json Display parent section only
  "styling": {
    "eyebrows": "section"
  }
  ```
</CodeGroup>

<div id="interaction-configuration">
  ## インタラクションの設定
</div>

`docs.json` の `interaction` プロパティで、ユーザーのナビゲーション要素とのインタラクションを制御します。

<div id="enable-auto-navigation-for-groups">
  ### グループの自動ナビゲーションを有効にする
</div>

ユーザーがナビゲーショングループを展開すると、テーマによっては自動的にグループ内の最初のページへ移動します。`drilldown` オプションでテーマの既定の動作を上書きできます:

```json
{
  "interaction": {
    "drilldown": true  // ユーザーがドロップダウンを展開したときに最初のページへ自動的に移動
  }
}
```

```json
{
  "interaction": {
    "drilldown": false // 移動は行わず、展開/折りたたみのみ行う
  }
}
```

* ナビゲーショングループが選択されたときに最初のページへ自動で移動させるには `true` を設定します。
* ナビゲーションを行わず、選択時はグループの展開/折りたたみのみ行うには `false` を設定します。
* 未設定の場合は、テーマの既定の動作が適用されます。

<div id="faq">
  ## FAQ
</div>

<Accordion title="避けるべきナビゲーションフォルダはどれですか？">
  ルートパス名（タイトルやフォルダ）として `/api` や `/mcp` を使用しないでください。`/api` および `/mcp` のパスは本番環境で予約されており、URL に `/api` または `/mcp` が含まれるページは 404 エラーになります。
</Accordion>
