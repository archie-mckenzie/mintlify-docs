`docs.json` ファイルを使うと、Markdown ファイルの集合を、ナビゲーション可能でカスタマイズされたドキュメントサイトに変換できます。必須のこの設定ファイルは、スタイル、ナビゲーション、各種連携などを制御します。ドキュメントの設計図だと考えてください。

`docs.json` の設定は、すべてのページに対してグローバルに適用されます。

<div id="setting-up-your-docsjson">
  ## `docs.json` の設定
</div>

はじめるには、`theme`、`name`、`colors.primary`、`navigation` を指定するだけで十分です。その他のフィールドは任意で、ドキュメントのニーズに応じて追加できます。

最適な編集体験のために、`docs.json` ファイルの先頭にスキーマ参照を記述してください。これにより、ほとんどのコードエディターでオートコンプリート、検証、ツールチップが有効になります：

```json
{
  "$schema": "https://mintlify.com/docs.json",
  "theme": "mint",
  "name": "Your Docs",
  "colors": {
    "primary": "#ff0000"
  },
  "navigation": {
    // ナビゲーション構造
  }
  // 残りの設定
}
```

<div id="customization">
  ### カスタマイズ
</div>

<ResponseField name="theme" required>
  サイトのレイアウトテーマ。

  次のいずれか: `mint`, `maple`, `palm`, `willow`, `linden`, `almond`, `aspen`。

  詳細は[Themes](ja/themes)を参照してください。
</ResponseField>

<ResponseField name="name" type="string" required>
  プロジェクト、組織、またはプロダクトの名称。
</ResponseField>

<ResponseField name="colors" type="object" required>
  ドキュメントで使用する配色。テーマによって適用方法が異なります。`primary` のみを指定した場合、すべてのカラー要素に適用されます。

  <Expandable title="Colors">
    <ResponseField name="primary" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$" required>
      ドキュメントのプライマリカラー。一般的にライトモードで強調に使用されます（テーマによって一部異なります）。

      `#` で始まる16進数のカラーコードである必要があります。
    </ResponseField>

    <ResponseField name="light" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
      ダークモードで強調に使用されるカラー。

      `#` で始まる16進数のカラーコードである必要があります。
    </ResponseField>

    <ResponseField name="dark" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
      ライト/ダーク両モードでボタンやホバー状態に使用されるカラー（テーマにより一部異なります）。

      `#` で始まる16進数のカラーコードである必要があります。
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="description" type="string">
  SEO および AI インデックス用のサイトの説明。
</ResponseField>

<ResponseField name="logo" type="string or object">
  ライトモードとダークモードの両方で使用するロゴ。

  <Expandable title="Logo">
    <ResponseField name="light" type="string" required>
      ライトモード用のロゴファイルへのパス。拡張子を含めてください。例: `/logo.png`
    </ResponseField>

    <ResponseField name="dark" type="string" required>
      ダークモード用のロゴファイルへのパス。拡張子を含めてください。例: `/logo-dark.png`
    </ResponseField>

    <ResponseField name="href" type="string (uri)">
      ロゴをクリックした際に遷移する URL。未指定の場合、ロゴはホームページにリンクされます。例: `https://mintlify.com`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string or object">
  拡張子を含む favicon ファイルへのパス。適切な favicon サイズに自動リサイズされます。
  単一ファイル、またはライト/ダークモード別のファイルを指定できます。例: `/favicon.png`

  <Expandable title="Favicon">
    <ResponseField name="light" type="string" required>
      ライトモード用の favicon ファイルへのパス。拡張子を含めてください。例: `/favicon.png`
    </ResponseField>

    <ResponseField name="dark" type="string" required>
      ダークモード用の favicon ファイルへのパス。拡張子を含めてください。例: `/favicon-dark.png`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="thumbnails" type="object">
  ソーシャルメディアやページプレビュー用のサムネイルのカスタマイズ。

  <Expandable title="Thumbnails">
    <ResponseField name="appearance" type="&#x22;light&#x22; | &#x22;dark&#x22;">
      サムネイルのビジュアルテーマ。未指定の場合、`colors` フィールドで定義されたサイトのカラースキームが使用されます。
    </ResponseField>

    <ResponseField name="background" type="string">
      サムネイルの背景画像。相対パスまたは絶対 URL を指定できます。
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="styling" type="object">
  ビジュアルスタイリングの設定。

  <Expandable title="Styling">
    <ResponseField name="eyebrows" type="&#x22;section&#x22; | &#x22;breadcrumbs&#x22;">
      ページのアイブローのスタイル。`section` を選ぶとセクション名、`breadcrumbs` を選ぶと完全なナビゲーションパスを表示します。デフォルトは `section`。
    </ResponseField>

    <ResponseField name="codeblocks" type="&#x22;system&#x22; | &#x22;dark&#x22;">
      コードブロックのテーマ。`system` を選ぶとサイトテーマに合わせ、`dark` は常にダークのコードブロックになります。デフォルトは `system`。
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="icons" type="object">
  アイコンライブラリの設定。

  <Expandable title="Icons">
    <ResponseField name="library" type="&#x22;fontawesome&#x22; | &#x22;lucide&#x22;" required>
      ドキュメント全体で使用するアイコンライブラリ。デフォルトは `fontawesome` です。

      <Note>
        ライブラリ設定に関わらず、個別のアイコンごとに、外部ホスティングされたアイコンのURL、プロジェクト内のアイコンファイルへのパス、または JSX 互換の SVG コードを指定できます。
      </Note>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="fonts" type="object">
  ドキュメントのフォント設定。デフォルトのフォントは [Inter](https://fonts.google.com/specimen/Inter) です。

  <Expandable title="Fonts">
    <ResponseField name="family" type="string" required>
      フォントファミリー（例: &quot;Open Sans&quot;, &quot;Playfair Display&quot;）。
    </ResponseField>

    <ResponseField name="weight" type="number">
      フォントの太さ（例: 400 や 700）。可変フォントは 550 のような細かなウェイトもサポートします。
    </ResponseField>

    <ResponseField name="source" type="string (uri)">
      フォントソースへの URL（例: https://mintlify-assets.b-cdn.net/fonts/Hubot-Sans.woff2）。Google フォントの `family` 名を指定した場合は [Google Fonts](https://fonts.google.com) が自動で読み込まれるため、ソース URL は不要です。
    </ResponseField>

    <ResponseField name="format" type="&#x22;woff&#x22; | &#x22;woff2&#x22;">
      フォントファイル形式。
    </ResponseField>

    <ResponseField name="heading" type="object">
      見出し用にフォント設定を個別に上書きします。

      <Expandable title="Heading">
        <ResponseField name="family" type="string" required>
          フォントファミリー（例: &quot;Open Sans&quot;, &quot;Playfair Display&quot;）。
        </ResponseField>

        <ResponseField name="weight" type="number">
          フォントの太さ（例: 400, 700）。可変フォントは 550 のような細かなウェイトもサポートします。
        </ResponseField>

        <ResponseField name="source" type="string (uri)">
          フォントソースへの URL（例: https://mintlify-assets.b-cdn.net/fonts/Hubot-Sans.woff2）。Google フォントの `family` 名を指定した場合は [Google Fonts](https://fonts.google.com) が自動で読み込まれるため、ソース URL は不要です。
        </ResponseField>

        <ResponseField name="format" type="&#x22;woff&#x22; | &#x22;woff2&#x22;">
          フォントファイル形式。
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="body" type="object">
      本文用にフォント設定を個別に上書きします。

      <Expandable title="Body">
        <ResponseField name="family" type="string" required>
          フォントファミリー（例: &quot;Open Sans&quot;, &quot;Playfair Display&quot;）。
        </ResponseField>

        <ResponseField name="weight" type="number">
          フォントの太さ（例: 400, 700）。可変フォントは 550 のような細かなウェイトもサポートします。
        </ResponseField>

        <ResponseField name="source" type="string (uri)">
          フォントソースへの URL（例: https://mintlify-assets.b-cdn.net/fonts/Hubot-Sans.woff2）。Google フォントの `family` 名を指定した場合は [Google Fonts](https://fonts.google.com) が自動で読み込まれるため、ソース URL は不要です。
        </ResponseField>

        <ResponseField name="format" type="&#x22;woff&#x22; | &#x22;woff2&#x22;">
          フォントファイル形式。
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="appearance" type="object">
  ライト/ダークモードのトグル設定。

  <Expandable title="Appearance">
    <ResponseField name="default" type="&#x22;system&#x22; | &#x22;light&#x22; | &#x22;dark&#x22;">
      デフォルトのテーマモード。ユーザーの OS 設定に合わせるには `system`、特定のモードを固定するには `light` または `dark` を選択します。デフォルトは `system` です。
    </ResponseField>

    <ResponseField name="strict" type="boolean">
      ライト/ダークモードのトグルを非表示にするかどうか。デフォルトは `true` です。
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="background" type="object">
  背景色と装飾の設定。

  <Expandable title="Background">
    <ResponseField name="image" type="string or object">
      サイトの背景画像。単一ファイル、またはライト/ダークモードで別々のファイルを指定できます。

      <Expandable title="Image">
        <ResponseField name="light" type="string" required>
          ライトモード用の背景画像のパス。拡張子を含めてください。例: `/background.png`。
        </ResponseField>

        <ResponseField name="dark" type="string" required>
          ダークモード用の背景画像のパス。拡張子を含めてください。例: `/background-dark.png`。
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="decoration" type="&#x22;gradient&#x22; | &#x22;grid&#x22; | &#x22;windows&#x22;">
      テーマの背景装飾。
    </ResponseField>

    <ResponseField name="color" type="object">
      ライト/ダークモード用のカスタム背景色。

      <Expandable title="Color">
        <ResponseField name="light" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
          ライトモード用の背景色。

          `#` で始まる16進数カラーコードである必要があります。
        </ResponseField>

        <ResponseField name="dark" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
          ダークモード用の背景色。

          `#` で始まる16進数カラーコードである必要があります。
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<div id="upgrading-from-mintjson">
  ## `mint.json` からのアップグレード
</div>

ドキュメントプロジェクトで非推奨の `mint.json` を使用している場合は、次の手順に従って `docs.json` へアップグレードしてください。

<Steps>
  <Step title="CLI をインストールまたは更新する">
    まだ [CLI](/ja/installation) をインストールしていない場合は、インストールしてください:

    <CodeGroup>
      ```bash npm
      npm i -g mint
      ```

      ```bash yarn
      yarn global add mint
      ```

      ```bash pnpm
      pnpm add -g mint
      ```
    </CodeGroup>

    すでに CLI をインストール済みの場合は、以下で最新化してください:

    ```bash
    mint update
    ```
  </Step>

  <Step title="docs.json ファイルを作成する">
    ドキュメントリポジトリで次を実行します:

    ```bash
    mint upgrade
    ```

    このコマンドは既存の `mint.json` から `docs.json` を作成します。生成されたファイルを確認し、すべての設定が正しいことを確認してください。
  </Step>

  <Step title="mint.json ファイルを削除する">
    `docs.json` の設定を確認できたら、古い `mint.json` は安全に削除できます。
  </Step>
</Steps>
