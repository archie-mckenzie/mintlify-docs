<Update label="August 24 - August 30" tags={["Improvements"]} rss={{ title: "Weekly Updates", description: "フィードバック収集の強化、ナビゲーションの改善、認証のバグ修正" }}>
  ## 主要リリース: フィードバック収集の強化

  * **大きな改善**: 読者は*thumbs up/down*を選択した後に、より詳細なフィードバック（選択式の項目や自由記述）を送信できるようになりました。コードブロックに対してもフィードバックを収集でき、すべての回答をダッシュボードのアナリティクスで確認できます。\
    *注: この機能をサイトで有効化するにはお問い合わせください。*

  ## ナビゲーションと利便性の改善

  * 末尾スラッシュや二重スラッシュを無視する改善により、docs.jsonで厳密に合わせる必要がなくなりました
  * `h1-6` のHTMLタグに `noAnchor` 属性を追加できるようになり、アンカーリンクの付与を防げます
  * Palm テーマに、Stripeに類似した左下に浮かぶ言語セレクターを追加
  * docs.json に新しいフィールド [`drilldown`](https://mintlify.com/docs/navigation#enable-auto-navigation-for-groups) を追加。ナビゲーショングループを展開した際に、そのグループの最初のページへ自動遷移させるかどうかを制御できます
  * 入れ子の番号付きリストが decimal/Roman と alpha のスタイルを交互に切り替えるように調整

  ## バグ修正と信頼性

  * ページ上にJSコンポーネントがある場合にスクロール位置のアンカーリンクが正しく動作しないバグを修正
  * `x-robots-tag noindex` ヘッダーが欠落していたために Google が生の `*.md` ファイルをインデックスしていたバグを修正
  * 保護されたドキュメントでの OAuth の不具合を修正。フロー完了後に開始ページへリダイレクトされない問題を解消
  * 認証保護されたドキュメントのプレビューで、ナビゲーションバー全体が表示されない問題を修正
  * 新しい画像CDNでのSVGの取り扱いに関するバグを修正

  ## コンポーネントとスタイリングの強化

  * `SidebarNavGroupDivider` 向けのカスタムスタイル用に新しいCSSセレクターを追加
  * セキュリティが定義された、MDXで定義されたAPIページ向けの新しいリグレッションテストを追加し、安定性をさらに向上

  ## パフォーマンス改善

  * KaTeX のCSSをcdnjsからCloudFront上の自社CDNへ移行し、レイテンシ削減によるパフォーマンス向上を実現
</Update>

<Update label="August 17 - August 23" tags={["Improvements"]} rss={{ title: "Weekly Updates", description: "画像レイアウトの修正、認証の強化、モバイルナビの改善、ウェブエディターの安定性向上" }}>
  ## 画像処理の改善

  * **大きな改善**: 幅と高さの属性が未指定でも、デフォルトで画像によるレイアウトシフトが発生しなくなりました。自動サイズ調整により、ページ読み込み中のコンテンツのジャンプを防止します
  * リポジトリ内のすべての静的ファイル（PDF、TXT、XML など）がデプロイ時に自動的にアップロード・配信され、アセットを完全にカバーします

  ## ウェブエディターとデプロイの改善

  * ウェブエディターでのブランチ作成ワークフローを修正し、新規作成ブランチへ正しく遷移して維持されるよう改善
  * マージコンフリクトダイアログを強化し、適切なエスケープ機能を実装。コンフリクトの解消にページリロードは不要に
  * 部分更新時に変更ページのみキャッシュを無効化するようにし、更新ワークフローのパフォーマンスを最適化してデプロイ時間を短縮

  ## 認証とナビゲーションの改善

  * カスタムサブディレクトリでの認証を新たにサポート。`https://yourdomain.com/docs` でドキュメントを提供している場合でも認証がシームレスに機能
  * リンクが1つだけ設定されている際にサイドバーが誤表示される不具合を修正
  * モバイルナビゲーションを包括的に刷新: ボタンを中央揃えにして適切なマージン/パディングを適用、ドロップダウンメニューのスペーシングを改善、空セクションの不要な区切り線とマージンを削除、Maple テーマの隙間/パディングの問題を修正

  ## コンポーネントとスタイリングの修正

  * `<h1-6>` タグが誤って Heading コンポーネントへ変換され、カスタムスタイリングに影響していた問題を解消
  * ダッシュボードにワンクリックのアシスタント設定トグルを追加し、管理を容易化

  ## 技術的改善と信頼性

  * 更新ワークフロー向けのログを強化し、デバッグと問題解決を高速化
  * 10以上の OpenAPI/AsyncAPI スペックを持つお客様向けに、個別ファイルの取得からリポジトリのクローンへ切り替え、GitHub のレート制限問題を解消
  * バックアップLLMのサポート、レート制限エラー処理の強化、検索ツールの堅牢性向上により、アシスタントの信頼性を改善
</Update>

<Update label="August 10 - August 16" tags={["Improvements"]} rss={{ title: "Weekly Updates", description: "パフォーマンス最適化、LLM 連携、ウェブエディターの磨き込み、API プレイグラウンドの強化" }}>
  ## パフォーマンスとビルドの最適化

  * MDX のトランスパイルは、未キャッシュの NextJS サーバーレス環境での各ページロード時ではなくデプロイ時に実行されるようになり、未キャッシュページの TTFB（最初のバイトまでの時間）が改善されました。
  * コンテンツベースのハッシュにより MDX に変更がない場合の再トランスパイルを防止し、大量ページを持つお客様の更新ワークフロー時間を約 50% 短縮（5 分超のデプロイはおおむね半減）
  * バックエンドでのデータベースインデックス追加とクエリの並列化により、ダッシュボードでのプレビューデプロイの閲覧が高速化
  * 各ページの `rsc` ペイロード内の重複した `navigation` データを排除してページサイズを削減—ページ数が多い場合やナビゲーション構造が複雑な場合に効果が顕著
  * さらに積極的なプリフェッチにより、即時ロードの発生頻度が向上

  ## APIプレイグラウンドと OpenAPI の強化

  * OpenAPI から MCP への変換をバックエンドに移行し、ホスト型 MCP サーバーにツールを含められるように（近日中により明確なドキュメントと設定オプションを提供予定）
  * APIプレイグラウンドに Ruby サポートを追加
  * 新機能として、[docs.json だけで API ページを指定](/ja/api-playground/openapi-setup#auto-populate-api-pages)できるようになりました。新たに mdx ファイルを作成する必要はありません。
  * OpenAPI スペック由来のドキュメントナビゲーションでの[`webhook` ページ](/ja/api-playground/openapi-setup#webhooks)をサポート
  * Anthropic、OpenAI、その他のプロバイダーへ遷移する際、マークダウンリンクからアンカー指定を除去して AI モデルのコンテキストを最適化

  ## ウェブエディターの改善

  * ファイルの作成/リネームは、Enter キーを押さずにクリックでフォーカスが外れた時点で保存
  * URL を特定のブランチに変更した際、意図した遷移先ではなく最後にアクティブだったブランチへリダイレクトされる問題を修正
  * `/` を含むブランチタイトルを正しく URL エンコードし、ナビゲーションの破損を防止
  * モノレポのダッシュボードエディターにおける `Ctrl+K` リンクショートカットが docs リポジトリのパスを前置して壊れたリンクを作成していた問題を修正

  ## アナリティクスと LLM 連携

  * カスタム `llms.txt` および `llms-full.txt` をサポート—ドキュメントリポジトリのルートに追加すると、`/llms.txt` および `/llms-full.txt` で配信され、LLM のカスタマイズが可能
  * [Hightouch のアナリティクス連携](/ja/integrations/analytics/hightouch#hightouch)を追加
  * コンテキストメニューのアナリティクストラッキングを強化（ダッシュボードでの可視化は近日公開）
  * 正しい配信を保証するため、`llms.txt` および `llms-full.txt` の e2e テストを追加

  ## コンポーネントとスタイリングの強化

  * カスタムの見出しスタイルを適用するため、`h{1-4}` タグでカスタムクラス名をサポート
  * カスタムページモードで `h{1-4}` がチップ付きの `Heading` コンポーネントとしてレンダリングされる問題を修正
  * カスタム CSS のターゲット指定のために [breadcrumbs](/ja/navigation#breadcrumbs) に CSS セレクターを追加
  * 高さ 56px で比率を維持するよう寸法を解析し、引き伸ばされていた Open Graph 画像を修正
  * コンテキストメニューで有効な場合の `VSCode` を `VS Code` に訂正
  * カスタムコンポーネント内の見出しがセマンティックな見出しとともに目次に表示される問題を修正

  ## バグ修正と信頼性

  * 生成に支障をきたす文字をサニタイズすることで、特定のページタイトルでの PDF レンダリング問題を修正
  * 空の OpenAPI JSON ファイルに遭遇した際の CLI エラー `Cannot convert undefined or null to object` を解消
  * 生成された Open Graph メタタグによりカスタムの `docs.json` が上書きされる問題を修正
  * アンカーリンクに遷移した場合の RSS フィードボタンのクリック挙動を、RSS リンクに origin + pathname を使用することで修正
  * ソースマップを削除して CLI のダウンロード速度を改善

  ## 技術的な改善

  * 早期のリグレッション検出のため、CI パイプラインにビジュアルテストを追加
  * エラーハンドリングとデバッグ機能を強化
  * 新機能およびエッジケースに対する包括的なテストカバレッジ
</Update>

<Update label="August 3 - August 9" tags={["Improvements"]} rss={{ title: "Weekly Updates", description: "Search API、OAuth の改善、アセット配信、MCP ツール群" }}>
  ## 認証の改善

  * グループ単位の公開アクセス：`docs.json` でページグループ全体を公開にでき、各ページに `public: true` を付ける必要がありません（[詳細](https://mintlify.com/docs/authentication-personalization/authentication-setup#group-level)）
  * 上流のクッキーを削除してサインアウトを完了するため、[OAuth 設定での `logoutURL` をサポート](https://mintlify.com/docs/authentication-personalization/authentication-setup#implementation-3)
  * OAuth エラー時は、指定した `logoutURL` へリダイレクトして認証フローを再開
  * コールバック前の OAuth/JWT フローで 500 エラーが一瞬表示される不具合を修正
  * 誤設定を防ぐため、OAuth/JWT 認証設定の URL から `https://` を自動で除去

  ## APIプレイグラウンドの強化

  * ドキュメント上でエージェントや MCP サーバーを構築できる新しい [Search API エンドポイント](https://mintlify.com/docs/api-reference/assistant/search)
  * `openapi` と `asyncapi` ファイルが指定パスで配信されるようになりました（例：`https://mydocsurl.extension/{openapi-or-file-name}.json`）
  * 生成フィールドの上書き、前文コンテンツのカスタマイズ、コードサンプル内のエンドポイント URL の変更のために、[OpenAPI ファイル内の `x-mint` フィールド](https://mintlify.com/docs/api-playground/openapi-setup#x-mint-extension)が利用可能に
  * OpenAPI 設定において、MCP ツールとして公開するルートを制御するため、[`x-mcp` は `x-mint.mcp` に変更](https://mintlify.com/docs/api-playground/openapi-setup#mcp)

  ## アシスタントのアップデート

  * 新しいメッセージがストリーミングされた際に、古いメッセージのアクションメニュー（コピーやサムズアップなど）が消える不具合を修正
  * 先週の[ホスト型 MCP サーバーのリリース](https://mintlify.com/docs/ai/model-context-protocol#accessing-your-mcp-server)後に発生した、ネストされた `/mcp/...` ページのアクセシビリティを修正

  ## パフォーマンスと信頼性

  * ドキュメントリポジトリ内のすべての画像・動画アセットが、ドメイン上の適切なパスで配信されるようになりました。たとえば、リポジトリに `/assets/marketing/my-logo.png` がある場合、`https://mydocsurl.extension/assets/marketing/my-logo.png` で利用可能です。
  * Mintlify ダッシュボードのログインでメールフィールドが自動フォーカスされ、すぐに入力開始可能に（使い勝手の改善）
  * ナビゲーション読み込みの高速化のため、Redis 上のカスタムドメインとサブドメインを最適化（約 50ms のレイテンシ削減）
  * 信頼性向上のため、PDF エクスポートにリトライロジックを追加
  * クッキー同意ポップアップが承認または却下後に再表示される問題を修正—最初の選択が尊重されるようになりました
  * Safari でページをクリップボードにコピーできない問題を、`navigator.write` に MIME の `type` を指定して修正

  ## 技術的な改善

  * Windows と pnpm 向けの CLI バグ修正に加え、リグレッション防止のための CI テストを追加
  * エラーログ出力を改善—デバッグ時のエンジニアリングチームの利便性が向上
  * `contentDirectory` ファイルがない場合の broken-link CI アクションに関する軽微な修正
  * 先週の認証保護付きプレビュー修正によるリグレッションで、ナビゲーション UI のアクティブタブが正しく設定されない問題を修正
  * テーマのライト背景色がアクティブタブのアイコンに適用されない問題を修正
  * ダッシュボードで認証タイプを変更すると、更新後に以前の保存タイプへ戻ってしまう問題を修正—保存後は新しい選択が保持されます
  * カスタム UI ライブラリを持つエンタープライズ顧客向けに内部 DX を改善—お客さまのコンポーネントを取り込み、短いタイムラインでの要望に対応しやすくなりました
</Update>
