<div id="about-mcp-servers">
  ## MCPサーバーについて
</div>

Model Context Protocol（MCP）は、ドキュメントなどの外部サービスとAIアプリケーション間の接続を標準化するオープンプロトコルです。Mintlifyは、あなたのドキュメントとOpenAPI仕様からMCPサーバーを生成し、Claude、Cursor、Gooseなど任意のMCPクライアントがドキュメントやAPIに接続できるよう、より広範なAIエコシステム向けにコンテンツを準備します。

あなたのMCPサーバーは、AIアプリケーションがドキュメントを検索し、APIとやり取りするためのツールを提供します。

<div id="accessing-your-mcp-server">
  ## MCPサーバーへのアクセス
</div>

<Note>
  MCPサーバーは公開ドキュメントに対してのみ生成されます。エンドユーザー認証が必要なドキュメントは、サーバー生成の対象外です。
</Note>

MCPサーバーは自動的に生成され、ドキュメントのURL配下の`/mcp`パスでホストされます。例えば、MintlifyのMCPサーバーは`https://mintlify.com/docs/mcp`で利用できます。

[MCPのダッシュボード](https://dashboard.mintlify.com/products/mcp)で、MCPサーバーのURLを確認してコピーできます。

<div id="configuring-your-mcp-server">
  ## MCPサーバーの設定
</div>

すべてのMCPサーバーにはデフォルトで`search`ツールが含まれており、他のツールからドキュメント内の情報をクエリできます。

[ProまたはEnterpriseプラン](https://mintlify.com/pricing?ref=mcp)をご利用の場合、OpenAPI仕様のエンドポイントをMCPツールとして公開できます。

エンドポイントをMCPツールとして公開するには、ファイルレベルまたはエンドポイントレベルで`x-mint`拡張内の`mcp`オブジェクトを使用します。たとえば、MintlifyのMCPサーバーには、Assistantチャットの作成、ステータスの取得、アップデートのトリガーを行うツールが含まれます。

MCPサーバーはセキュリティファーストのアプローチに従っており、APIエンドポイントはデフォルトでは公開されません。エンドポイントをMCPツールとして利用可能にするには、明示的に有効化する必要があります。AIツール経由での公開に適した、安全なエンドポイントのみを公開してください。

<div id="file-level-configuration">
  ### ファイル単位の設定
</div>

OpenAPI 仕様ファイルで、デフォルトで全エンドポイントに MCP を有効化し、必要に応じて特定のエンドポイントを除外します:

```json
{
  "openapi": "3.1.0",
  "x-mint": {
    "mcp": {
      "enabled": true
    }
  },
  // ...
  "paths": {
    "/api/v1/users": {
      "get": {
        "x-mint": {
          "mcp": {
            "enabled": false // このエンドポイントでは MCP を無効化
          }
        },
        // ...
      }
    }
  }
}
```

<div id="endpoint-level-configuration">
  ### エンドポイント単位の設定
</div>

特定のエンドポイントで MCP を有効にします:

```json
{
  "paths": {
    "/api/v1/users": {
      "get": {
        "x-mint": {
          "mcp": {
            "enabled": true,
            "name": "get-users",
            "description": "Get a list of users"
          },
          // ...
        }
      }
    },
    "/api/v1/delete": {
      "delete": {
        // `x-mint: mcp` がないため、このエンドポイントは MCP ツールとして公開されません
        // ...
      }
    }
  }
}
```

<div id="using-your-mcp-server">
  ## MCPサーバーの利用
</div>

ユーザーは、利用中のAIツールにあなたのMCPサーバーを接続する必要があります。

1. MCPサーバーのURLを公開します。
2. ユーザーがMCPサーバーのURLをコピーして、自身のツールに追加します。
3. ユーザーはツール経由であなたのドキュメントとAPIエンドポイントにアクセスします。

ユーザーがあなたのMCPサーバーに接続できるよう支援する方法の例は次のとおりです:

<Tabs>
  <Tab title="コンテキストメニュー">
    ドキュメントの任意のページからユーザーが MCP サーバーに接続できるよう、[コンテキストメニュー](/ja/ai/contextual-menu)にオプションを追加します。

    | オプション | 識別子 | 説明 |
    | :----- | :--------- | :---------- |
    | **MCP サーバー URL をコピー** | `mcp` | ユーザーのクリップボードに MCP サーバー URL をコピーします。 |
    | **Cursor に接続** | `cursor` | MCP サーバーを Cursor にインストールします。 |
    | **VS Code に接続** | `vscode` | MCP サーバーを VS Code にインストールします。 |
  </Tab>

  <Tab title="Claude">
    <Steps>
      <Step title="MCP サーバー URL を取得します。">
        [ダッシュボード](https://dashboard.mintlify.com/products/mcp)に移動し、MCP サーバー URL を確認します。
      </Step>

      <Step title="ユーザー向けに MCP サーバー URL を公開します。">
        ユーザー向けのガイドを作成し、MCP サーバー URL と Claude への接続手順を記載します。

        1. Claude の設定の [Connectors](https://claude.ai/settings/connectors) ページに移動します。
        2. **Add custom connector** を選択します。
        3. MCP サーバー名と URL を追加します。
        4. **Add** を選択します。
        5. Claude 使用時に、添付ボタン（プラスアイコン）を選択します。
        6. MCP サーバーを選択します。
      </Step>
    </Steps>

    詳細は [Model Context Protocol のドキュメント](https://modelcontextprotocol.io/docs/tutorials/use-remote-mcp-server#connecting-to-a-remote-mcp-server)をご覧ください。
  </Tab>

  <Tab title="Claude Code">
    <Steps>
      <Step title="MCP サーバー URL を取得します。">
        [ダッシュボード](https://dashboard.mintlify.com/products/mcp)に移動し、MCP サーバー URL を確認します。
      </Step>

      <Step title="ユーザー向けに MCP サーバー URL を公開します。">
        ユーザー向けのガイドを作成し、MCP サーバー URL と Claude Code への接続コマンドを記載します。

        ```bash
        claude mcp add --transport http <name> <url>
        ```
      </Step>
    </Steps>

    詳細は [Claude Code のドキュメント](https://docs.anthropic.com/en/docs/claude-code/mcp#installing-mcp-servers)をご覧ください。
  </Tab>

  <Tab title="Cursor">
    <Steps>
      <Step title="MCP サーバー URL を取得します。">
        [ダッシュボード](https://dashboard.mintlify.com/products/mcp)に移動し、MCP サーバー URL を確認します。
      </Step>

      <Step title="ユーザー向けに MCP サーバー URL を公開します。">
        ユーザー向けのガイドを作成し、MCP サーバー URL と Cursor への接続手順を記載します。

        1. コマンドパレットを開くには <kbd>Command</kbd> + <kbd>Shift</kbd> + <kbd>P</kbd>（Windows は <kbd>Ctrl</kbd> + <kbd>Shift</kbd> + <kbd>P</kbd>）を押します。
        2. &quot;Open MCP settings&quot; を検索します。
        3. **Add custom MCP** を選択します。`mcp.json` ファイルが開きます。
        4. `mcp.json` でサーバーを設定します:

        ```json
        {
          "mcpServers": {
            "<your-mcp-server-name>": {
              "url": "<your-mcp-server-url>"
            }
          }
        }
        ```
      </Step>
    </Steps>

    詳細は [Cursor のドキュメント](https://docs.cursor.com/en/context/mcp#installing-mcp-servers)をご覧ください。
  </Tab>

  <Tab title="VS Code">
    <Steps>
      <Step title="MCP サーバー URL を取得します。">
        [ダッシュボード](https://dashboard.mintlify.com/products/mcp)に移動し、MCP サーバー URL を確認します。
      </Step>

      <Step title="ユーザー向けに MCP サーバー URL を公開します。">
        ユーザー向けのガイドを作成し、MCP サーバー URL と VS Code への接続手順を記載します。

        1. `.vscode/mcp.json` ファイルを作成します。
        2. `mcp.json` でサーバーを設定します:

        ```json
        {
          "servers": {
            "<your-mcp-server-name>": {
              "type": "http",
              "url": "<your-mcp-server-url>"
            }
          }
        }
        ```
      </Step>
    </Steps>

    詳細は [VS Code のドキュメント](https://code.visualstudio.com/docs/copilot/chat/mcp-servers)をご覧ください。
  </Tab>
</Tabs>

<div id="example-connecting-to-the-mintlify-mcp-server">
  ### 例: Mintlify MCP サーバーへの接続
</div>

Mintlify MCP サーバーに接続して、Mintlify API とやり取りし、ドキュメントを検索します。これにより、ローカル環境での Mintlify の使い方に関するより正確な回答が得られ、ユーザーを自分たちの MCP サーバーに接続させる方法の参考にもなります。

<Tabs>
  <Tab title="コンテキストメニュー">
    このページ上部のコンテキストメニューを開き、**Connect to Cursor** または **Connect to VS Code** を選んで、任意の IDE に Mintlify MCP サーバーを接続します。
  </Tab>

  <Tab title="Claude">
    Mintlify MCP サーバーを Claude で使用するには:

    <Steps>
      <Step title="Mintlify MCP サーバーを Claude に追加する">
        1. Claude の設定内の [Connectors](https://claude.ai/settings/connectors) ページに移動します。
        2. **Add custom connector** を選択します。
        3. Mintlify MCP サーバーを追加します:

        * Name: `Mintlify`
        * URL: `https://mintlify.com/docs/mcp`

        4. **Add** を選択します。
      </Step>

      <Step title="チャットで MCP サーバーにアクセスする">
        1. Claude 利用中に、添付ボタン（プラスアイコン）を選択します。
        2. Mintlify MCP サーバーを選択します。
        3. Mintlify について Claude に質問します。
      </Step>
    </Steps>

    詳細は [Model Context Protocol のドキュメント](https://modelcontextprotocol.io/docs/tutorials/use-remote-mcp-server#connecting-to-a-remote-mcp-server)を参照してください。
  </Tab>

  <Tab title="Claude Code">
    Mintlify MCP サーバーを Claude Code で使用するには、次のコマンドを実行します:

    ```bash
    claude mcp add --transport http Mintlify https://mintlify.com/docs/mcp
    ```

    次を実行して接続をテストします:

    ```bash
    claude mcp list
    ```

    詳細は [Claude Code のドキュメント](https://docs.anthropic.com/en/docs/claude-code/mcp#installing-mcp-servers)を参照してください。
  </Tab>

  <Tab title="Cursor">
    <PreviewButton href="cursor://anysphere.cursor-deeplink/mcp/install?name=mintlify&config=eyJ1cmwiOiJodHRwczovL21pbnRsaWZ5LmNvbS9kb2NzL21jcCJ9">Cursor にインストール</PreviewButton>

    Mintlify MCP サーバーを Cursor に接続するには、**Install in Cursor** ボタンをクリックします。手動で MCP サーバーに接続する場合は、次の手順に従ってください:

    <Steps>
      <Step title="MCP 設定を開く">
        1. コマンドパレットを開くには <kbd>Command</kbd> + <kbd>Shift</kbd> + <kbd>P</kbd>（Windows では <kbd>Ctrl</kbd> + <kbd>Shift</kbd> + <kbd>P</kbd>）を使用します。
        2. &quot;Open MCP settings&quot; を検索します。
        3. **Add custom MCP** を選択します。これにより `mcp.json` ファイルが開きます。
      </Step>

      <Step title="Mintlify MCP サーバーを設定する">
        `mcp.json` に次を追加します:

        ```json
        {
          "mcpServers": {
            "Mintlify": {
              "url": "https://mintlify.com/docs/mcp"
            }
          }
        }
        ```
      </Step>

      <Step title="接続をテストする">
        Cursor のチャットで &quot;What tools do you have available?&quot; と尋ねます。Cursor に Mintlify MCP サーバーが使用可能なツールとして表示されるはずです。
      </Step>
    </Steps>

    詳細は Cursor のドキュメントの [Installing MCP servers](https://docs.cursor.com/en/context/mcp#installing-mcp-servers) を参照してください。
  </Tab>

  <Tab title="VS Code">
    <PreviewButton href="https://vscode.dev/redirect/mcp/install?name=mintlify&config=%7B%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Fmintlify.com%2Fdocs%2Fmcp%22%7D">VS Code にインストール</PreviewButton>

    Mintlify MCP サーバーを VS Code に接続するには、**Install in VS Code** ボタンをクリックします。手動で MCP サーバーに接続する場合は、`.vscode/mcp.json` ファイルを作成して、次を追加します:

    ```json
    {
      "servers": {
        "Mintlify": {
          "type": "http",
          "url": "https://mintlify.com/docs/mcp"
        }
      }
    }
    ```

    詳細は [VS Code のドキュメント](https://code.visualstudio.com/docs/copilot/chat/mcp-servers) を参照してください。
  </Tab>

<div id="authentication">
  ## 認証
</div>

MCP で API エンドポイントを有効化すると、サーバーは OpenAPI の `securitySchemes` と `securityRequirement` で定義された認証要件を反映します。各種キーはツール側で直接扱われ、Mintlify によって保存・処理されることはありません。

ユーザーが保護されたエンドポイントの呼び出しを AI ツールに指示した場合、ツールはその時点で必要な認証情報の入力をユーザーに求めます。

<div id="troubleshooting">
  ## トラブルシューティング
</div>

<AccordionGroup>
  <Accordion title="MCPサーバーに検索ツールしか表示されない">
    OpenAPI仕様があるにもかかわらず、MCPサーバーが検索ツールのみを公開している場合:

    1. OpenAPI仕様が正しく、かつアクセス可能であることを確認してください。
    2. 対象のエンドポイントで `x-mint.mcp.enabled: true` を設定し、MCPを明示的に有効化していることを確認してください。
    3. デプロイのログでOpenAPI処理エラーがないか確認してください。

    OpenAPIの処理に失敗した場合でも、機能を維持するためにサーバーは検索ツールのみで動作を継続します。
  </Accordion>

  <Accordion title="認証に関する問題">
    ユーザーから認証の問題が報告された場合:

    1. OpenAPI仕様に適切な `securitySchemes` の定義が含まれていることを確認してください。
    2. 有効化したエンドポイントが、指定した認証方式で正常に動作することを確認してください。
  </Accordion>

  <Accordion title="ツールの説明が不足している／不明瞭">
    AIツールがAPIエンドポイントを効果的に活用できていない場合:

    1. エンドポイントに詳細な `summary` と `description` フィールドを追加してください。
    2. パラメーター名と説明が自己説明的であることを確認してください。
    3. MCPダッシュボードで、エンドポイントがツールとしてどのように表示されるかを確認してください。
  </Accordion>
</AccordionGroup>
