---
title: "シングルサインオン（SSO）"
description: "チームの管理ダッシュボードへのログイン方法をカスタマイズする"
---

<Info>
  SSO 機能は [Enterprise プラン](https://mintlify.com/pricing?ref=sso)で利用可能です。
</Info>

SAML と OIDC を使ってダッシュボードにシングルサインオンできます。Okta、Google Workspace、Microsoft Entra を利用中の場合は、各プロバイダー向けの設定ドキュメントをご用意しています。その他のプロバイダーをご利用の場合は、[お問い合わせください](mailto:support@mintlify.com)。

<div id="okta">
  ## Okta
</div>

<Tabs>
    <Tab title="SAML">
        <Steps>
            <Step title="アプリケーションを作成">
                `Applications` から、SAML 2.0 を使用した新しいアプリ統合を作成します。
            </Step>
            <Step title="統合を設定">
                次を入力します：
                * Single sign-on URL（Mintlify 提供）
                * Audience URI（Mintlify 提供）
                * Name ID Format: `EmailAddress`
                * Attribute Statements:
                    | Name | Name format | Value
                    | ---- | ----------- | -----
                    | `firstName` | Basic | `user.firstName` |
                    | `lastName` | Basic | `user.lastName` |
            </Step>
            <Step title="IdP 情報を送信">
                アプリケーションの設定が完了したら、Sign-On タブに移動してメタデータ URL をお送りください。
                いただいた情報を用いて、こちらで接続を有効化します。
            </Step>
        </Steps>
    </Tab>
    <Tab title="OIDC">
        <Steps>
            <Step title="アプリケーションを作成">
                `Applications` から、OIDC を使用した新しいアプリ統合を作成します。
                アプリケーションタイプは `Web Application` を選択してください。
            </Step>
            <Step title="統合を設定">
                Authorization Code グラントタイプを選択し、Mintlify 提供の Redirect URI を入力します。
            </Step>
            <Step title="IdP 情報を送信">
                アプリケーションの設定が完了したら、General タブに移動して client ID と client secret を確認します。
                これらに加えて Okta インスタンスの URL（例：`<your-tenant-name>.okta.com`）を安全にご提供ください。1Password や SendSafely などのサービス経由で送付できます。
            </Step>
        </Steps>
    </Tab>
</Tabs>

<div id="google-workspace">
  ## Google Workspace
</div>

<Tabs>
    <Tab title="SAML">
        <Steps>
            <Step title="アプリケーションを作成する">
                `Web and mobile apps` で、`Add app` ドロップダウンから `Add custom SAML app` を選択します。  
                <Frame>
                    ![](/images/gsuite-add-custom-saml-app.png)
                </Frame>              
            </Step>
            <Step title="IdP 情報を送信する">
                表示される SSO URL、Entity ID、x509 証明書をコピーし、Mintlify チームに送付してください。
                <Frame>
                    ![](/images/gsuite-saml-metadata.png)                    
                </Frame>
            </Step>
            <Step title="連携を設定する">
                Service provider details ページで、次を入力します:
                * ACS URL（Mintlify 提供）
                * Entity ID（Mintlify 提供）
                * Name ID format: `EMAIL`
                * Name ID: `Basic Information > Primary email`

                <Frame>
                    ![](/images/gsuite-sp-details.png)
                </Frame>

                次のページで、以下の属性ステートメントを入力します:
                | Google Directory Attribute | App Attribute |
                | -------------------------- | ------------- |
                | `First name` | `firstName`  |
                | `Last name` | `lastName` |

                以上が完了し、ユーザーをアプリケーションに割り当てたら、チームまでご連絡ください。弊社側でお客様のアカウントの SSO を有効化します。
            </Step>
        </Steps>
    </Tab>
</Tabs>

<div id="microsoft-entra">
  ## Microsoft Entra
</div>

<Tabs>
    <Tab title="SAML">
        <Steps>
            <Step title="アプリケーションを作成する">
                1. 「Enterprise applications」で **New application** を選択します。
                2. **Create your own application** を選び、「Integrate any other application you don't find in the gallery (Non-gallery).」を選択します。
            </Step>
            <Step title="SAML を設定する">
                Single Sign-On のセットアップページに移動し、**SAML** を選択します。「Basic SAML Configuration」で次の値を入力します:
                * Identifier (Entity ID): Mintlify が提供する Audience URI
                * Reply URL (Assertion Consumer Service URL): Mintlify が提供する ACS URL
                
                それ以外の値は空のままにして、**Save** を選択します。
            </Step>
            <Step title="Attributes & Claims を設定する">
                Attributes & Claims セクションを編集します:
                1. 「Required Claim」の **Unique User Identifier (Name ID)** を選択します。
                2. Source attribute を `user.primaryauthoritativeemail` に変更します。
                3. Additional claims で、次のクレームを作成します:
                    | Name | Value |
                    | ---- | ----- |
                    | `firstName` | `user.givenname` |
                    | `lastName` | `user.surname` |
            </Step>
            <Step title="Mintlify に IdP 情報を送信する">
                アプリケーションのセットアップが完了したら、「SAML Certificates」セクションに移動し、App Federation Metadata URL を送付してください。
                この情報をもとに、当方で接続を有効化します。
            </Step>
            <Step title="ユーザーを割り当てる">
                Entra アプリケーションの「Users and groups」に移動し、ダッシュボードへアクセスさせるユーザーを追加します。
            </Step>
        </Steps>
    </Tab>
</Tabs>
